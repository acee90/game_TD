# 레퍼런스 — LoL 아레나 · TFT 증강 기능 분류

> 상태: **현행** · 최종 갱신: 2026-07-14
> [증강 전수 목록](lol-arena-tft-augments-v1.0.md)(패치 16.13)의 501종을 **기능별로 분류**한 문서.
> 목록 문서가 원본 덤프라면, 이 문서는 **우리가 붙인 해석**이다 — 원본에는 이런 분류가 없다.
> 목적: 우리 증강 72종([명세 §7](../design/game-system-spec.md))에 어떤 축이 비어 있는지 본다.

---

## 1. 분류 기준과 방법

**한 증강에 primary 카테고리 1개**를 붙였다. 겹치면 아래 우선순위로 하나만 고른다.

| key | 카테고리 | 정의 |
|---|---|---|
| `stat` | **스탯 강화** | 순수 수치 증가. 새 행동 없음 |
| `combat-effect` | **특수 능력 추가** | 새 발동 효과·행동 (온히트, 처형, 소환, 장판…) |
| `ability-mod` | **스킬 개조** | 기존 스킬·궁극기의 작동을 바꾼다 |
| `economy` | **경제** | 골드·아이템·유닛 지급, 비용 감소, 이자 |
| `synergy` | **시너지·특성** | 특성/계열/조합 조건에 걸린 것 |
| `sustain` | **생존** | 회복·보호막·부활·피해 감소 |
| `tempo` | **템포·자원** | 경험치·레벨·리롤·부활 시간 등 전투 외 회전 |
| `scaling` | **성장·퀘스트** | 누적·조건 달성으로 전투력이 계속 커진다 |
| `utility` | **유틸·기동** | 이동속도·CC·시야 등 |

우선순위(위가 이긴다): `synergy` → `ability-mod` → `tempo` → `scaling` → `sustain` → `combat-effect` → `utility` → `stat` → `economy`.

두 가지 규칙이 결과를 크게 좌우한다:

- **`stat`은 순수 수치일 때만.** 애매하면 stat이 아니다.
- **`economy`는 지급이 효과의 *전부*일 때만.** 유닛·아이템을 주면서 전투·특성 효과까지 주면 **그 효과로 분류하고** `grant` 태그를 단다. 지급은 전달 수단으로 본다. (예: "제이스를 획득하고, 제이스의 스킬이 바뀐다" → `ability-mod` + `grant`)

보조 태그(중복 허용): `conditional`(조건부) · `stacking`(중첩) · `risk`(대가 있음) · `team`(아군 대상) · `aoe`(광역) · `unit-grant`(유닛 지급) · `grant`(자원 지급이 딸려 있음).

### 이 분류의 한계 — 읽기 전에

- **원본 데이터에 없는 정보다.** 라이엇은 증강에 기능 태그를 붙여 배포하지 않는다. 이 분류는 위 기준표를 주고 LLM이 501종의 설명문을 읽어 붙인 것이며, 사람이 전수 검수하지 않았다.
- 따라서 **경계 사례는 흔들린다.** 예를 들어 '받는 피해 20% 감소'는 수치만 보면 `stat`이지만 이 기준표에서는 `sustain`으로 간다. 카테고리별 **정확한 개수보다 비율의 크기 차이**를 보는 용도다.
- **`economy` 규칙은 한 번 고쳤다.** 처음엔 '주면 무조건 economy'로 뒀더니 TFT 275종 중 162종(59%)이 economy로 쏠려 TFT가 경제 게임처럼 보였다. 실제로는 '유닛을 주고 그 유닛의 스킬을 바꾸는' 형태가 많았다. 위 규칙으로 그 200종을 재판정한 결과가 이 문서의 수치다.
- 아레나 `rarity=4` 25종은 등급 정체가 미확인이지만(목록 문서 §2.4), 분류에는 포함했다.
