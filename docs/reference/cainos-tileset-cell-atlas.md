## 1. 레이어 모델 [원본확정: 공식 문서]

- **높이 레벨 하나 = 소팅 레이어 하나.** 같은 레벨 안에서는 Order in Layer(= 화면 Y)로 정렬.
- 벽 타일은 두 레벨 사이를 잇는 조각으로, 높은 레벨의 가장자리에 붙는다.
- 레벨 전환은 계단(TX Struct) + 트리거로 처리(Unity 전용 세부는 우리와 무관).
- map-lab 대응: `elev`(0=허공/1=지면/2=plateau) + `depth = 화면행·100 + 오프셋` — 동일 모델.

## 2. TX Tileset Wall (512×512, 32px 셀 16×16) — Frame A 해부

좌상단 **Frame A**가 벽 문법의 원형: **시트 셀 (col 1–3, row 1–4)에 정확히 정렬된 3×4 템플릿**
(픽셀 스캔 실측: x=32..127, y=32..159).

```
        col1        col2        col3
row1  NW rim      N rim       NE rim      ← 상단 rim(두께 8px, 셀 상단에 위치)
row2  W rim       (투명)      E rim       ← 측면 rim(두께 ~11px), 내부는 투명=잔디가 비친다
row3  SW 캡+face  캡+face상단  SE 캡+face  ← 남벽 위칸: 기둥머리 캡(18px) + 벽돌 시작
row4  SW face+발단 face+발단   SE face+발단 ← 남벽 아래칸: 벽돌 + 밝은 돌 발단(8px)
```

### 남쪽 벽 = 정확히 2셀 스택 [원본확정: 열 스캔 x=75]

| y(시트) | 내용 |
|---|---|
| 96–113 | 캡 상판 + 전면 베벨 그라데이션 (**기둥 위**) |
| 114–150 | 벽돌 face (캡보다 어두운 톤, 줄눈 주기 ~9px) |
| 151–158 | 밝은 돌 발단 (**기둥 아래**) |

- **face를 세로로 반복하지 않는다** — row3+row4 두 장이 전부. (더 높은 벽이 필요하면
  시트의 별도 2×2 face 블록을 쓰지만, 레퍼런스 씬의 옹벽은 전부 2셀이다.)
- row4는 아랫칸(지면/허공)의 자리를 차지한다 → plateau 남벽은 지면 1타일을 소비한다.
- 측면 rim의 블록 이음매는 **주기 16px, 위상 y=44 기준** — 32px 크롭이 이음매에 걸리면
  체인이 점선이 된다. (한 번 틀렸던 부분. 셀 그리드 그대로 자르면 자연히 맞는다.)
- 그 외: Frame B(내부용 프레임: face가 북쪽에 있는 방 형태), 세로 스트립 2종(부서진 기둥 장식),
  십자 문장 조각, 2×2 face 변형 2종(고벽용).

## 3. TX Tileset Grass (256×256)

- **상단 절반**: 32px 잔디 타일 — 좌측 4×4 = 풀포기 변형, 우측 4×4 = 꽃(흰/파/노) 변형.
- **하단 절반**: **16px 서브셀 단위** 포장돌 (32px 타일당 돌 4개). 돌은 16k 격자에 정렬
  [원본확정: 행/열 스캔 — 세로 몸통 y∈16k..16k+14, 가로 x∈16k+1..16k+14].
  - 좌측 블록(셀 0–3, 8–15행대): 깨끗한 돌 / 우측 상단 행: 풀 덮인 마모 돌 /
    우측 하단: 낱개·부서진 돌.
  - **오토타일이 아니다** — 돌 하나하나가 완결된 둥근 사각. 군집은 "인접 배치"로만 만든다.

## 4. 기타 시트

- **TX Tileset Stone Ground**: 대형 석재 플랫폼(캡 프레임)·통로 포장 — 현재 미사용(path.png,
  platform.png로 일부 기추출). 세부 셀 분석 미실시(미확인).
- **TX Struct**: 독립 벽 세그먼트(기둥 양끝, 깨끗 3 + 파손 3), 아치 게이트 2, 난간 계단 4.
  계단이 레벨 전환 조각. 우리 맵에는 아직 미사용.
- **TX Plant / TX Props (+with Shadow)**: 나무 3(96~120px≈3셀), 덤불, 항아리·통·바위,
  석상, 기둥 스택, 원형 단상 등. Extra 폴더 합본은 그림자 포함이라 이걸 쓴다.

## 5. Scene Overview 역산 (근사 실측)

- 텍스처 셀 = **32px**, 확대 없음. 밀도는 셀을 키워서가 아니라 **경로 굴곡·소품·돌 군집**으로 낸다.
- 높이 레벨: 지면 + plateau 2단(부분적으로 3단), 옹벽은 전부 위 2셀 문법.
- 보행로: 폭 2셀 내외, 16px 돌을 유기적 군집으로 포장(가장자리는 마모·낱개 돌).
- 소품 비율: 나무 = 3셀급 랜드마크. 10×10셀 구역당 나무 1–2 + 소형 소품 2–4 수준.
- 시사점: **비주얼 셀을 2배(128px)로 키우면 소품/타일 비율이 무너진다** →
  논리 1셀 = 64px(32px 타일 2×2)이 상한. (2026-07-24 확대 실험 롤백 근거.)

## 6. Scene Overview 층별 판독 (오버레이 이미지)

- **판독 이미지**: [cainos-scene-layer-reading.png](cainos-scene-layer-reading.png)
  (상세: [cainos-scene-layer-reading-detail.png](cainos-scene-layer-reading-detail.png))
- 범례: 초록=L1 기저 지면 / 보라=L2 테라스 / 빨강=L3 최상단(분수·석상 플랫폼) /
  어두움=허공. 흰 선=32px 셀, 노란 선=4셀(128px).
- 격자 위상 [원본확정: 포장돌 16px 격자·rim 경계 픽셀 스캔]: 씬은 **(0,0) 원점
  32px 격자에 스냅**, 확대 없는 1x 텍셀. 층 틴트 사각형은 육안 판독(정밀도 ±1셀).
- 구조: 3단 테라스(L1→L2→L3) + 허공. 모든 층 경계는 §2의 2셀 벽 스택,
  레벨 전환은 계단(TX Struct). 최외곽도 L1→허공 벽으로 동일 문법.
- **핵심 결론: 밀도는 셀 확대가 아니라 "층 수"에서 나온다.** 37×37셀 소형 씬이
  3단 단차로 어디서나 벽·계단·단차가 시야에 걸린다. 셀을 키우면(2×2 확대 실험)
  소품/타일 비율만 무너진다 — §5와 함께 확대 롤백의 근거.
