## 0. 한 줄 요약

**Legacy 효과는 단일 타격과 조합의 식별력을 거의 높이지 못하면서, 혼잡 구간의 잔상에서는
큰 흰색 플래시로 유닛과 경로를 가렸다.** 평타 Legacy 레이어 제거를 우선 실험하고,
일반 조합·GOD 조합은 별도의 카툰형 시그니처 효과로 다시 만들어야 한다.

이 결론은 자동 캡처의 1차 관찰이며 사람 플레이테스트 결정은 아니다.

## 1. 캡처 조건

| 항목 | 값 |
|---|---|
| Unity | 6000.5.3f1, URP 17.5 |
| 해상도 | Game View 952×562 |
| 모드 | `FullLegacy`, `CodeOnly` |
| 시나리오 | 일반 타격, 광역 타격, 일반 조합, GOD 조합, 보스 사망, 혼잡 전투 |
| 시점 | PRE, PEAK, SETTLE |
| 결과 | 2모드 × 6시나리오 × 3시점 = 36장 |
| 출력 | `unity/Captures/VfxReview/baseline-20260717/` — 생성물이라 Git 제외 |

`FullLegacy`는 코드 생성 버스트·링·트레일에 `Resources/Fx/*`의 임포트 프리팹을 더한
현행 조건이다. `CodeOnly`는 임포트 프리팹만 끄며 전투 판정과 코드 생성 효과는 유지한다.

## 2. 관찰

### 2.1 혼잡 전투에서 Legacy 잔상이 가독성을 해친다

`FullLegacy / Crowd / SETTLE`은 교차로와 세로 경로에 큰 흰색 플래시가 중첩됐다.
같은 프레임의 `CodeOnly`는 적, 투사체, 타워와 경로가 계속 보였다. 현재 팩을 유지하더라도
반복 평타에 임포트 머즐·착탄을 매번 재생하는 방식은 우선 제거 대상이다.

### 2.2 단일 타격과 광역 타격의 두 조건 차이가 작다

전체 게임 카메라에서 일반 타격과 광역 타격의 `FullLegacy`/`CodeOnly` 차이는 작았다.
임포트 레이어가 매우 짧거나 작아서 보상은 적고, 여러 발이 겹칠 때만 밝기가 누적되는
구조로 보인다. 이는 더 강한 팩을 바로 추가하기보다 반복 효과의 레이어·크기·타이밍을
먼저 정리해야 한다는 근거다.

### 2.3 일반 조합과 GOD 조합의 가치 차이가 부족하다

샘플 프레임에서 일반 조합과 GOD 조합은 새 타워의 외형 변화 외에 명확한 실루엣 차이가
약했다. 게임의 기저 리듬인 일반 조합과 잭팟인 GOD 조합을 Legacy 팩 선택으로 해결하지 않고,
같은 문법을 공유하는 두 단계의 자체 시그니처 효과로 제작할 필요가 있다.

### 2.4 보스 사망도 전체 화면 기준으로 작다

보스 사망의 피크와 잔상은 발생 위치를 찾은 뒤에는 보이지만, L4 이벤트로 즉시 시선을
옮길 정도의 실루엣은 아니었다. 일반 적 사망을 단순 확대하는 접근 대신 큰 파편·보상색·
짧은 카메라 반응을 하나의 묶음으로 다시 검증해야 한다.

## 3. 1차 판정

| 항목 | 판정 | 다음 실험 |
|---|---|---|
| 반복 평타 Legacy 머즐·착탄 | 제거 우선 | `CodeOnly`를 기본으로 실제 10초 전투 A/B |
| 광역 Legacy 폭발 | 보류 | 범위 정보와 폭발 장식을 분리한 뒤 재촬영 |
| 일반 조합 | 재제작 | 수축 → 각진 파편 → 티어 링 → 스케일 팝 |
| GOD 조합 | 재제작 | 일반 조합 문법 + 금색 수직 형태·룬·제한적 히트스톱 |
| 보스 사망 | 재제작 | 큰 질량 변화와 보상색 중심의 L4 실루엣 |
| 새 Particle Pack 구매 | 아직 보류 | P1 노이즈 제거 후 무료/상용 후보를 같은 하네스로 비교 |

## 4. 한계와 다음 측정

- `VFX OFF` 조건은 아직 없다. 코드 효과 자체의 기여도는 다음 배치에서 분리한다.
- 혼잡 시나리오는 결정적 합성 상태이며 실제 후반 10초 플레이가 아니다.
- 정지 프레임은 타이밍과 움직임의 질을 완전히 평가하지 못한다. 다음에는 짧은 영상도 남긴다.
- Profiler·Frame Debugger 수치는 아직 기록하지 않았다.
- 실행 중 URP가 memoryless depth surface load/store 메시지 2건을 남겼다. 이번 비교의
  시각 결과와 직접 연결되는지는 미확인이다.
- 최종 채택은 사람 검수와 실제 플레이 가독성 테스트 뒤에만 한다.

## 5. 재현

1. Unity에서 Play Mode에 진입한다.
2. Unity MCP `execute_code`로 아래를 실행한다.

```csharp
return GodTD.View.VfxReviewHarness.StartBaselineBatch("baseline-YYYYMMDD");
```

3. 진행 상태는 아래로 확인한다.

```csharp
return GodTD.View.VfxReviewHarness.Status();
```

하네스는 실행 중에만 백그라운드 재생을 활성화하고 완료 시 원래 설정으로 복원한다.
캡처 생성물은 `.gitignore`의 `unity/Captures/` 규칙으로 제외한다.
