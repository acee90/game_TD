# 설계 — 영웅 성장 통합: 골드 → XP → 레벨업 스탯 선택

> 상태: **초안** (2안 채택 여부 확인 대기) · 작성일 2026-07-11
> 범위: `web/src/data/hero.ts` · `game/hero.ts` → 테스트 → 명세 → Unity 미러.
> 배경: [명세 §12 0.5](../design/game-system-spec.md) 영웅 파워커브 과속,
> [income-curve.csv](../balance/income-curve.csv) 기준 밸런싱.

## 0. 문제

현행 영웅 파워 = 스탯(골드) × **레벨 배수(XP, 1+2.4×(L-1))** × 증강.

- 레벨 배수가 **골드 없이** 킬 XP만으로 오른다 — Lv17에 민첩 2개만 사도 DPS ~985
  (Lv3 타워의 6배). 골드 저울(income-curve)에 영웅이 올라가 있지 않다.
- 골드로 XP도 사게 하면(TFT식) 스탯 구매와 **골드 싱크 2개가 병렬**이 되어 과복잡.

## 1. 두 안 비교 (플레이테스트 2026-07-11 제안)

| | 1안: 레벨 한도 → 만렙 후 스탯 구매 | 2안: 레벨 무한 → 레벨업마다 스탯 선택 |
|---|---|---|
| 축 개수 | 2개가 **순차**로 남음 (XP기 → 스탯기) | **1개로 통합** (골드→XP→레벨→스탯) |
| 초반 빌드 표현 | 만렙 전엔 스탯 결정 없음 — 죽은 구간 | 첫 레벨업부터 힘/민/지 선택 |
| 증강 스케줄(레벨 연동) | 한도와 충돌 — 재설계 필요 | 그대로 동작 |
| 전환점 | 만렙 순간 경제가 갑자기 바뀜 | 없음 (연속) |

**권고: 2안.** 1안은 시스템을 합치는 게 아니라 시간순으로 배열할 뿐이고,
만렙 전 구간에서 빌드 선택(힘/민/지)이 사라진다.

## 2. 2안 구체 설계 (구현 시 기준)

- **XP 소스**: ① 골드 구매(주축, TFT식 — 버튼 1개, 비용은 XP 곡선이 흡수)
  ② 킬 XP(부축 — 막타 ×2 유지하되 총량 축소. 지금의 XP_PER_MOB 0.65 → 추가 하향).
- **레벨 배수(1+2.4×(L-1)) 폐지.** 파워는 레벨업마다 고르는 스탯 포인트 × 증강만.
  → 파워커브 과속의 근본 제거. `statGrant`(20회마다 +1) 같은 성장 곡선은
  레벨당 지급 포인트로 이식.
- **레벨업 = 힘/민첩/지능 중 택1** (+N포인트). UI는 증강 카드식 일시정지가 아니라
  비차단 선택(기본값: 마지막 선택 반복) — 레벨이 잦으면 클릭 스팸이 된다.
- **레벨 속도 확실히 감속** (사용자 지시): 목표 = R45(성장 정지)에 Lv 25~30 도달,
  라운드당 ~0.6레벨. 현행 xpToNext 지수(14×1.06^L) 유지하되 킬 XP 축소 +
  구매 골드가 주 연료가 되도록 역산 — income-curve 저점의 20~30%를 영웅에 넣는
  플레이어 기준.
- **밸런스 앵커**: 같은 골드의 한계 DPS가 영웅 ≈ 타워 (income-curve 기준, ±30%).
  영웅 프리미엄(이동·어그로)은 30% 이내 가격에 반영.
