## 0. 결론

Legacy Particle Pack 안에서 프리팹을 더 고르는 것만으로는 최종 품질을 만들지 않는다.
현행 화면은 스타일라이즈드 로우폴리 중세 판타지인데, Legacy 팩의 사실적인 불꽃·플라즈마
플립북과 코드 생성 링·원반·스파크가 한 타격에 겹쳐 형태 언어와 중요도 계층이 무너진다.

다음 혼합 전략을 실험한다.

1. **기존 중복 연출을 먼저 줄여** 애셋 문제와 연출 문제를 분리한다.
2. 무료 카툰 VFX를 **로컬 비교 실험에만** 사용해 방향을 검증한다.
3. 평타·일반 사망처럼 반복되는 효과는 하나의 경량 스타일 라이브러리로 통일한다.
4. 게임 정체성인 **일반 조합·GOD 조합·보스·대표 영웅 스킬은 자체 제작**한다.
5. 후반 웨이브의 성능과 실제 플레이 가독성을 통과한 뒤에만 현행 기준으로 승격한다.

상용 팩을 선택해야 한다면 스타일 적합성은 `POLYGON Particle FX`가 우선 후보이고,
더 그래픽한 카툰 타격감을 원하면 `Cartoon FX Remaster`가 대안이다. 단 이 저장소는 공개이므로
두 팩 모두 **원본을 커밋하지 않고 로컬 실험 브랜치에서만 비교**한다(§6).

## 1. 현행 진단

### 1.1 구현 구조

현재 [`GameViewFx.cs`](../../unity/Assets/Scripts/View/GameViewFx.cs)는 다음을 동시에 사용한다.

- 코드 생성 `ParticleSystem` 버스트
- 코드 생성 투사체·`TrailRenderer`
- 코드 생성 펄스·링·광역 범위 원판
- `Resources/Fx/*`의 Legacy 팩 프리팹 4종
- 직접 구현한 풀링·동시 인스턴스 상한

현재 한 번의 행동에 붙는 레이어:

| 행동 | 현행 연출 |
|---|---|
| 투사체 발사 | 코드 버스트 + Legacy 머즐 플래시 + 구체/볼트 + 트레일 |
| 단일 착탄 | 코드 버스트 + Legacy 스파크 |
| 광역 착탄 | 범위 원판 + 시그니처 링/펄스/버스트 + Legacy 폭발/플라즈마 |
| 일반 적 사망 | 코드 버스트 |
| 조합/GOD | 코드 버스트 + 펄스 + 스케일 팝 + 선택적 히트스톱 |
| 보스 사망 | 대형 버스트 2회 + 펄스 + 흔들림 + 히트스톱 |

`FxKind`는 의미가 아니라 `Shot.Color`의 HEX 문자열로 추론한다. 색을 팔레트 조정하면
이펙트 종류가 바뀔 수 있고, 같은 색을 쓰는 서로 다른 행동을 구분하기 어렵다.

### 1.2 어색함의 원인 가설

아직 플레이테스트로 확정하지 않은 가설이다.

1. **형태 불일치** — 평면색 로우폴리 월드와 사실적인 반투명 플립북이 충돌한다.
2. **중복 레이어** — 작은 평타도 코드 FX와 임포트 FX를 함께 재생해 시각 밀도가 과하다.
3. **중요도 역전** — 판당 반복되는 평타가 조합·스킬과 비슷하게 밝고 크다.
4. **색 중심 구분** — 실루엣과 움직임보다 색에 의존해 색이 겹치면 의미도 겹친다.
5. **투명 오버드로우** — 다중 서브 파티클과 Bloom이 후반 웨이브에서 겹칠 위험이 있다.

검증은 §8의 VFX OFF / 코드 FX / 후보 팩 세 조건 비교로 한다.

## 2. 목표 아트 방향 — 카툰 판타지 토이 디오라마

이 이름은 실행을 위한 임시 레이블이며 새 세계관 결정이 아니다. 현행
[worldbuilding §3·§7](../design/worldbuilding.md#3-표현-범위)의 기준을 VFX로 번역한다.

### 2.1 형태 언어

- 원형의 부드러운 연기보다 **삼각형·육각형·별·굵은 파편·각진 퍼프**를 쓴다.
- 하나의 효과는 큰 형태 1개와 보조 형태 1~2개로 읽혀야 한다.
- 사진형 연기·불 텍스처, 미세 불티 수십 개, 긴 반투명 잔류 연기는 피한다.
- 범위 정보는 바닥의 얇은 도형, 타격 장식은 위로 솟는 메시로 분리한다.

### 2.2 색·재질

- 기본은 평면색 또는 2단 음영. 알파 그라데이션보다 명확한 외곽을 우선한다.
- 계열색 1개 + 밝은 코어 1개 + 어두운 보조색 1개의 최대 3색 구조를 기본으로 한다.
- HDR·Bloom은 중심 코어에 제한하고 평타보다 조합·스킬·보스에 강하게 배분한다.
- 마법 효과가 화면 전체를 네온으로 바꾸지 않도록 철·목재·석재 월드 팔레트와 함께 본다.

### 2.3 움직임

각 효과는 다음 세 박자를 공유한다. 시간값은 시작 가설이며 플레이테스트로 조정한다.

1. **예고** — 방향·범위를 읽게 하는 짧은 수축/모임
2. **타격** — 한 프레임에 가장 큰 실루엣과 가장 밝은 코어
3. **회수** — 작은 파편·잔상만 남기고 빠르게 사라짐

힘은 파티클 개수보다 크기 변화, 속도 대비, 카메라 반응, 음향과의 동기로 만든다.

## 3. 연출 중요도 계층

| 등급 | 이벤트 | 원칙 |
|---|---|---|
| L0 정보 | 선택 범위, 이동 목적지, 스킬 실제 범위 | 장식보다 정확도. 얇고 낮게, Bloom 없음 |
| L1 반복 | 일반 평타, 일반 착탄, 일반 적 사망 | 1~2 레이어, 짧은 수명, 카메라 반응 없음 |
| L2 보상 | 일반 조합, 레벨업, 증강 획득 | 명확한 실루엣과 색. 주변 평타보다 한 단계 강함 |
| L3 전술 | 영웅 스킬, 광역 타격, 보스 등장 | 예고와 타격을 분리. 실제 범위가 읽혀야 함 |
| L4 잭팟 | GOD 조합, 보스 처치 | 제한적 흔들림·히트스톱·대형 형태. 동시 발생 시 최우선 |

평타를 더 화려하게 만드는 것보다 L1을 억제해 L2~L4가 상대적으로 강해지게 하는 것을
우선한다.

## 4. 먼저 제작할 버티컬 슬라이스 6종

### V1. 일반 조합

두 유닛이 중심으로 수축 → 굵은 파편 → 티어색 링 → 새 유닛 스케일 팝.
P1의 기저 리듬이므로 첫 번째 커스텀 시그니처로 만든다.

### V2. GOD 조합

일반 조합 문법을 유지하되 금색 수직 형태와 왕관/룬 실루엣을 추가한다.
짧은 히트스톱과 카메라 반응은 유지하되 화면 전체 흰 플래시는 사용하지 않는다.

### V3. 일반 평타

작은 메시 투사체 + 짧은 트레일 + 착탄 파편 2~4개만 남긴다.
매 발 머즐 플래시는 기본적으로 끄고, 포병 또는 상위 티어처럼 발사 실루엣이 중요한 경우만 쓴다.

### V4. 화염/광역 스킬

실제 범위 텔레그래프와 폭발 장식을 별도 레이어로 만든다. 범위는 낮고 얇게,
폭발은 위로 솟는 각진 화염 메시와 파편으로 표현한다.

### V5. 보스 등장·사망

등장은 바닥 그림자/균열과 큰 질량 변화, 사망은 일반 적과 다른 대형 파편과 보상 색을 쓴다.
일반 적 사망 버스트를 단순 확대하지 않는다.

### V6. 증강 획득

전투 월드보다 UI 중심으로 제작한다. 등급 테두리, 짧은 Sweep, 선택 카드의 색이 영웅에게
전달되는 연출을 한 묶음으로 검증한다.

## 5. 기술 선택

현재 프로젝트는 Unity 6000.5.3f1, URP 17.5다.

### 5.1 기본 — Particle System + Shader Graph

대부분의 효과는 Built-in Particle System(Shuriken)으로 제작한다. 작은 전투 효과를
자주 풀링하는 현재 구조와 맞고, 공식 문서도 Built-in Particle System과 VFX Graph를
용도에 따라 구분한다.

- [Unity 6.5 Particle effects](https://docs.unity3d.com/6000.5/Documentation/Manual/ParticleSystems.html)
- [Choosing your particle system solution](https://docs.unity3d.com/6000.4/Documentation/Manual/ChoosingYourParticleSystem.html)

공유 Shader Graph는 최소 두 개로 시작한다.

- `VfxStylizedUnlit` — 평면색, HDR 코어, 알파 클립/디더, 색상 주입
- `VfxStylizedAdditive` — 제한적 발광, 짧은 트레일과 마법 코어

각진 불꽃·연기·파편은 빌보드만 쓰지 않고 저폴리 메시 파티클을 섞는다.

### 5.2 VFX Graph — 초기에는 도입하지 않음

VFX Graph는 대규모 GPU 효과에 유리하지만 현재 `manifest.json`의 직접 의존성이 아니며,
이 게임의 우선 문제는 파티클 수가 아니라 스타일과 계층이다. 다음 조건이 생길 때만 검토한다.

- GOD/최종 보스 효과가 Particle System 예산으로 구현되지 않음
- 한 효과에 매우 많은 GPU 파티클이 필요함
- 프로파일링에서 CPU 파티클 시뮬레이션이 실제 병목으로 확인됨

### 5.3 제작 도구

| 목적 | 도구 |
|---|---|
| 저폴리 파편·퍼프·룬 메시 | Blender |
| 마스크·플립북·그라데이션 | Krita / Affinity Photo / Photoshop 중 보유 도구 |
| 머티리얼 | Unity Shader Graph |
| 재생·풀링·변형 | Unity Particle System + `UnityEngine.Pool.ObjectPool<T>` |
| 성능 | Unity Profiler · Frame Debugger · 필요 시 Profile Analyzer |
| 스타일 비교 | 동일 카메라·동일 시드 10초 캡처 |

새 범용 피드백 프레임워크나 Addressables는 이 단계에서 추가하지 않는다. 현재 흔들림·히트스톱과
풀링이 이미 있으므로 의미 기반 카탈로그로 정리한 뒤 필요성을 다시 판단한다.

## 6. 외부 애셋 후보와 라이선스 경계

[worldbuilding §8](../design/worldbuilding.md#8-외부-애셋-정책)에 따라 공개 저장소에 커밋하는
외부 애셋은 CC0만 허용한다. Unity Asset Store 에셋은 무료여도 원본 파일을 공개 저장소에
재배포하지 않는다.

| 후보 | 용도 | 판정 |
|---|---|---|
| 현행 Legacy Particle Pack | 중복 제거 A/B, 기능 폴백 | 최종 스타일 기준으로 사용하지 않음. 현재 커밋 상태와 배포 권한 별도 감사 필요 |
| [Cartoon FX Remaster Free](https://assetstore.unity.com/packages/vfx/particles/cartoon-fx-remaster-free-109565) | 무료 스타일 적합성 비교 | 로컬/비공개 실험 전용. 프리팹 그대로 채택 금지 |
| [Cartoon FX 1 Remaster](https://jeanmoreno.com/unity/cartoonfx/) | 그래픽한 타격·마법 후보 | 로컬/비공개 실험 전용. 카툰성이 목표와 맞을 때만 구매 검토 |
| [POLYGON Particle FX](https://syntystore.com/products/polygon-particle-fx-pack) | 로우폴리 평타·파편·마법 후보 | 스타일 1순위 비교 후보. Unity 6.5/URP 머티리얼을 별도 브랜치에서 검증 |
| 자체 제작 + CC0 입력 | 조합·GOD·보스·대표 스킬 | 공개 저장소의 최종 경로 |

상용 팩의 효과를 보고 비슷한 원본 리소스를 복제하지 않는다. 추상적인 색·타이밍·형태 원칙만
비교하고, 최종 자체 제작물은 자체 메시·텍스처·셰이더로 만든다.

## 7. 구현 구조 개편

### 7.1 색상 추론 제거

현행 `Classify(string hex)`를 의미 기반 큐로 대체한다.

```text
VfxCue
├─ BasicHit
├─ SplashHit
├─ Merge
├─ MergeGod
├─ SkillFire
├─ SkillArcane
├─ SkillWind
├─ BossSpawn
└─ BossDeath
        ↓
VfxCatalog (ScriptableObject)
        ↓
VfxPlayer + ObjectPool
        ↓
Prefab / palette / scale / priority / camera response
```

`VfxCue`가 전투 판정에 영향을 주지 않도록 View 이벤트 DTO에 둔다. 불가피하게 `Shot`에
표현 필드를 추가하면 `web/src`와 Unity Core 미러를 함께 갱신하되, 판정 로직과 분리한다.

### 7.2 `VfxCatalog`

큐마다 다음을 데이터로 둔다.

- 프리팹 또는 코드 생성 효과
- 기본 스케일·수명·색상 변형 허용 여부
- 중요도 L0~L4와 동시 재생 상한
- 흔들림·히트스톱 요청값
- 저사양 대체 큐

프리팹 경로 문자열을 코드에 흩뿌리지 않고 Inspector에서 한 카탈로그로 확인하게 한다.

### 7.3 풀링

현행 커스텀 풀은 버티컬 슬라이스 동안 유지해 범위를 줄인다. 카탈로그가 안정된 뒤
`UnityEngine.Pool.ObjectPool<T>`로 통일하고, 파티클 루트가 아니라 모든 서브 시스템의
종료 여부를 기준으로 회수한다.

## 8. 실행 단계와 통과 조건

### P0. 기준선 캡처·애셋 감사

진행 기록: [Unity 전투 VFX 기준선 비교 v0.1](../reports/unity-vfx-baseline-v0.1.md)에서
`코드 FX + Legacy`와 `코드 FX만`의 6개 결정적 시나리오, 3개 시점을 캡처했다.
혼잡 구간의 Legacy 잔상이 가독성을 해치는 현상은 확인했지만 `VFX OFF`, 실제 후반 10초,
성능·라이선스 감사까지 끝나지 않았으므로 P0 통과 전이다.

- [ ] 같은 시드/카메라로 현행 전투 10초와 후반 혼잡 구간을 캡처
- [ ] VFX OFF / 코드 FX만 / 코드+Legacy 세 조건 캡처
- [ ] `EffectExamples`와 `Resources/Fx`의 출처·라이선스·공개 저장소 커밋 가능 여부 확인
- [ ] Profiler·Frame Debugger 기준선 기록

**통과:** 어색함이 애셋 스타일, 중복 레이어, Bloom/스케일 중 어디에서 오는지 관찰 기록이 있다.

### P1. 노이즈 제거

- [ ] 평타의 Legacy 머즐·착탄을 끌 수 있는 실험 토글
- [ ] 광역의 범위 정보와 장식 레이어 분리
- [ ] 평타 L1, 조합 L2, 스킬 L3, GOD/보스 L4의 상대 크기 적용

**통과:** VFX를 줄였는데도 타격 인지가 유지되고 조합·스킬이 더 잘 보인다.

### P2. 버티컬 슬라이스

- [ ] §4의 V1~V6 제작
- [ ] 공통 셰이더와 팔레트 프리셋 적용
- [ ] 후보 팩은 로컬 비교용으로만 A/B

**통과:** UI를 가리고 5초 영상을 보여도 관찰자가 평타/조합/스킬/보스를 구분하고,
세계와 같은 스타일이라고 평가한다.

### P3. 구조화

- [ ] `VfxCue`와 `VfxCatalog`
- [ ] 풀링·우선순위·동시 재생 상한
- [ ] 반복 평타가 L3/L4 효과를 가리지 않도록 우선순위 드롭

**통과:** 색상 변경이 효과 종류를 바꾸지 않고, 프리팹 교체가 게임 판정 코드 수정 없이 가능하다.

### P4. 전체 확장

순서: 조합/GOD → 평타/착탄 → 영웅 스킬 → 보스 → 일반 적 사망/누출 → 레벨업/부활 → 환경.

**통과:** 모든 이벤트가 §2의 색·형태·움직임 규칙과 §3 중요도 계층을 따른다.

### P5. 성능·플레이테스트

측정값은 목표 PC에서 기준선을 잡은 뒤 확정한다. 시작 가설:

- 1080p 목표 PC에서 후반 혼잡 구간 60fps 유지
- 95퍼센타일 프레임 시간이 16.7ms 안쪽
- VFX ON/OFF GPU 차이는 우선 1.5ms 이내를 실험 목표로 두고 실측 후 조정
- 전체 화면을 덮는 투명 레이어가 지속되지 않음
- 파티클을 꺼도 판정과 시뮬레이션 결과가 동일함

정성 테스트 질문:

1. 무엇이 조합됐는지 즉시 알았는가?
2. GOD과 일반 조합의 가치 차이가 느껴지는가?
3. 평타가 스킬 범위나 적 진행을 가리는가?
4. 자동 시전된 스킬의 원인과 결과를 연결할 수 있는가?
5. 효과가 월드 모델·UI와 같은 게임처럼 보이는가?

## 9. 문서 승격 규칙
