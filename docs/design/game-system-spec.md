## 0.5 코어 루프 — 플레이어가 반복하는 경험

```
웨이브 예측 → 유닛 생성·조합 → 경제 배분 → 전투 관찰 → 보스 소환 판단
     → 증강·레벨 보상 → 빌드 강화 → 다음 웨이브 (25초, R50+ 30초 주기)
```

| 단계 | 보는 정보 | 내리는 결정 | 비용 | 즉시 피드백 |
|---|---|---|---|---|
| 웨이브 예측 | 라운드 번호, 사이클 위치(5R 주기), 몹 장갑 계단 | 티어를 올릴 때인가 | — | 다음 웨이브의 체감 난이도 |
| 유닛 생성·조합 | 빈 타일, 보유 유닛 구성, **현재 생성가** | 어디에 몇 기 뽑나 | 미네랄 12+ (누적 증가 §3) | 랜덤 유닛 등장, 조합 시 상위 티어 팝 |
| 경제 배분 | 미네랄/가스 잔고, 프로브 수 | 유닛 vs 프로브 vs **영웅 XP** / 업글 vs 리롤 vs 스킬 개조 | 미네랄·가스 | 버튼 비용 갱신, 가스 수급률 |
| 전투 관찰 | 몹 진행, 영웅 어그로, 타워 화력 | 영웅을 어디 세울까 | — | 피해 숫자, 누출 경고 |
| 보스 소환 | 쿨타임, 열린 레벨, 현재 화력 | 지금 걸어볼 만한가 (저지 불가 리스크) | 실패 시 라이프 2+Lv | 처치 보상 or 누출 |
| 증강 보상 | 카드 3장, 보유 계열, 등급 | 몰 것인가 넓힐 것인가, 리롤할 것인가 | 리롤 가스 | 시너지 발동, 빌드 방향 확정 |

실패에서 배워야 하는 것: 누출 위치(화력 공백 구간), 보스 실패(소환 타이밍 오판),
영웅 사망(스탯 대비 과한 어그로).

## 1. 맵과 경로

- **십자(cross) 지형**: 중앙 제단 타일 1 + 팔 4방향. 타워 타일 **41개**
  (십자 17 = 중앙 1 + 팔당 4 × 4방향, 모서리 4구역 × 6).
- **몹 경로**: 북측 왼쪽 입구에서 스폰 → 십자 외곽선을 **반시계** 일주 → 북측 오른쪽
  출구. 웨이포인트 14개, 경로 위 위치는 좌표가 아니라 "일주 거리" 스칼라 하나로 표현.
- **제단**: 게임 시작과 함께 십자 중앙에 무료 제공. 그 타일에는 타워 불가.
- 영웅과 몹은 경로 위로만 다닌다. 빈 곳 클릭 시 최근접 경로점으로 투영 이동.
- **경로 폭과 몹 겹침 분리** (2026-07-19): 보행 반폭 12 → **20px**. 몹은
  (진행도, 횡오프셋) 2D 원으로 취급해 겹치면 서로 밀어낸다(소프트 분리).
  멈춘 몹·보스는 진행 방향으로 안 밀리고 횡으로만 비켜, 몹끼리 길을 완전히
  막는 교착이 없다. **전투 판정은 여전히 진행도 1D** — 분리는 공간 점유·표시만.

### 2.1 난이도 곡선 — 구간별 직접 지정

```
웨이브 총체력 = R1 686.4
              × 1.190573 (R2~15) × 1.17 (R16~29)
              × 1.16 (R30~40) × 1.17 (R41~50) × 1.10 (R51+)
              — R40 총체력 363,750을 기준점으로 고정
몹 1기 체력  = 총체력 ÷ 몹 수          (사이클 경계에서 수가 리셋 → 개당 체력 점프)
몹 장갑      = floor(라운드/5) × 1.1   (5라운드 계단 — 유지)
몹 공격력    = 0.5 + 0.3×라운드 (사냥꾼 웨이브 ×6, 영웅·허수아비만 피격)
수입 모델    = 시작 + Σ웨이브보상 + 킬 미션 + 보스 중간값 (결정론·연속 보간)
              — 소득 설계·검산 전용 시트(income-curve.csv). 곡선과는 분리됐다.
```

- **R40 앵커 재조형** (2026-07-21 플레이테스트): 적절하다고 확인한 R40 총체력 363,750은
  유지한다. R1은 20% 높이고 R16~29는 17%, R30~40은 16%로 직접 지정한다. 남은 R2~15는
  R40에 정확히 도달하도록 19.0573%로 역산한다.
- **접촉 공격력 절반** (2026-07-21): 체력은 타워 화력 검증, 공격력은 탱킹 압력이라는 역할을
  분리한다. 기본/라운드 계수를 1/0.6→0.5/0.3으로 낮추고 사냥꾼 ×6 정체성은 유지한다.
- **시뮬 대신 시트**: 모델이 결정론이라 [income-curve.csv](../balance/income-curve.csv)가
  라운드별 수입·DPS·총체력·성장률을 코드에서 그대로 뽑는다 — 상수를 바꾸면 시트로
  즉시 검산한다. 시뮬레이션은 검산용이지 기준이 아니다.
- 남긴 교훈: 벽 구간만은 지수여야 한다 — 선형 벽은 상대 성장률이 감소해 보드가
  결국 이긴다(봇 R138 생존 사고).
- 사이클 리듬은 유지: 몹 수가 사이클 안에서 늘고(11→15), 경계에서 리셋되며 개당 체력이
  뛴다 — "새 몹은 적지만 굵다".
- **누출**: 몹이 출구에 도달하면 라이프 -1, 미네랄 +5 [원본확정]. 시작 라이프 20.

## 3. 유닛(타워)과 조합

- **생성**: `12 + 0.45×max(0, n−8)` 미네랄 (n = 누적 생성 횟수 — 조합해도 리셋 안 됨),
  랜덤 유닛 (4종족 × 티어 풀, 7종 균등) [프로토 — 원본 기본가 12는 strings:412].
  고정 12는 GOD 20기·타일 포화를 만들었다(§11).
- **조합**: 같은 이름 2기가 모이면 **자동으로** 다음 티어 랜덤 유닛 1기로 합성.
  연쇄 조합 지원. 이것이 이 게임의 기저 도파민이다(판당 ~69회).
- **티어**: 5단계 (Lv1~4 + **GOD**). 티어 피해 배수 [1, 3, 9, 28, 95] × 기본 7.
  사거리 [120, 140, 160, 185, 225]. 기본 공격 간격 0.9초.
- **태그**: power(피해 1.6×, 사거리 1.15×, 보스 우선 타겟) / splash(피해 0.7×, 광역) /
  speed(간격 0.5×). 스플래시는 폭심에서 멀수록 **3단계 계단 감쇠**(안쪽 ⅓ 100% ·
  가운데 75% · 바깥 50%, 2026-07-19) — 영웅 스플래시도 동일.
- **크리쳐 종족**: 보조 타워 — 피해 0.6×이지만 사거리 안 몹 감속.
  감속은 **가장 강한 것 하나만** 적용(도배 방지). 티어별 [0.9, 0.84, 0.76, 0.66, 0.5].
- **판매** 가능. GOD 풀은 보스 처치 수에 따라 조기/확장 분기.

## 4. 경제 — 재화마다 싱크가 경쟁한다

```
미네랄 → 유닛(12+0.45×(n−8)) / 프로브(100×1.5ⁿ, 최대 16기) / 영웅 XP(20골드=20XP)
가스   → 종족 업그레이드(8+4n+n²) / 증강 리롤(12→24, 최대 2회) / 스킬 개조(30×1.35ⁿ)
```

라운드별 수입 저점·고점과 몬스터 압력·기대 화력의 공통 기준표:
**[income-curve.csv](../balance/income-curve.csv)** (`gen:balance` 산출).

**설계 원칙**: 싱크가 하나뿐인 재화는 자동 소비가 되어 선택이 사라진다.
(개편 전 GA가 전 세대 "프로브 8 + 가스 즉시 업글"로 수렴했던 이유.)

### 4.1 소득 — 보스가 주축 (2026-07-19 개편, 사용자 지시)

| 소득원 | 값 |
|---|---|
| 시작 자원 | 미네랄 55 · 가스 6 [원본확정] |
| 웨이브 클리어 | **20 + 0.2×(라운드−5)** — 거의 평탄한 기본기 [프로토] |
| (참고) 보스 소환 쿨타임 | **한 바퀴(56.2초)의 80% = 45초** — 유도값 (2026-07-19) |
| 보스 처치 | Lv1~8: 5 / 10 / 18 / 32 / 55 / 90 / 150 / 248 [프로토] — ×2~4 배증안은 롤백(GOD 조기화) |
| 킬 미션 | **20킬마다 +20** (반복, 무한) [프로토 — 원본 반복 20킬 보상과 같은 리듬] |
| 누출 위로금 | +5 [원본확정] |
| 가스 | 프로브 1기당 0.4/초 |

설계 의도: 옛 구조는 라운드 보상이 지수-이차로 커져 R60에 보스 수입의 4배가 됐고
소환 실패 리스크가 무의미했다. 이제 라운드 보상·킬 미션은 깔아두는 기본기(합계
라운드당 ~50)이고, **소득 성장은 보스 사다리에서만 나온다** — 중간값 모델 기준
중후반 보스 비중 **~52%**(개편 전 20~38%).

보스 보상 자체를 ×2~4로 올리는 안은 **시도했다가 롤백**했다 — 총 수입이 부풀어
GOD 1기 도달이 R14 → R11로 당겨졌다. 비중 목표는 라운드 보상을 낮추는 것만으로
달성되고, 그 편이 **총량(=GOD 페이스)을 안 건드린다**. 웨이브 체력은 성장률 직접
지정(§2.1)이라 어느 쪽이든 난이도 곡선은 흔들리지 않는다.

**GOD 페이스 참고** (실제 조합 로직 3,000판 몬테카를로): GOD 1기 = 유닛 생성 중앙
**52기 · 910골드**(이론 최소 16기 — 같은 이름 2기 조합이라 짝 못 만난 잔여가 평균
10.9기 깔린다). 2기 73기 · 3기 91기 · 5기 125기. 전액 타워 투자 가정 시 도달 상한은
R14 / R20 / R25 / R38.

### 4.2 프로브 딜레마

비용 **60 + 30n 선형** (2026-07-19, 지수 ×1.5 폐지 — 사용자 지시). 지수는 10기째가
2,300을 넘어 가스 엔진이 중반에 봉인됐다 — 파워 사슬(가스→업그레이드 선형→피해 선형)의
시작이 지수로 잠기는 모순. 첫 두 기(60·90)는 지수 시절과 같아 초반 앵커 불변,
16기 총액 4,560으로 후반의 실질 선택지가 된다.
(옛 이력: 30×1.3ⁿ은 "무조건 프로브 1순위" 지배 전략 → 100×1.5ⁿ → 60×1.5ⁿ → 선형.)

### 4.3 종족 업그레이드

가스로 종족별 공격력 +10% 복리. 비용 8+4n+n². 타워 화력의 후반 축.

## 5. 보스 — 상시 소환, 리스크 있는 결정

- **소환**: 라운드와 무관하게 언제든, 비용 0. 쿨타임은 **보스 한 바퀴의 80%로 유도**
  (2026-07-19, 사용자 지시): 경로 1,460px ÷ 속도 26 = 56.2초 × 0.8 = **45초**.
  고정 상수가 아니라 유도값이라 경로·속도를 바꿔도 "앞 보스가 한 바퀴를 거의 돌 무렵
  다음을 부를 수 있다"는 의미가 유지된다. 80%라 소환은 늘 조금씩 겹친다 — 빨리 못
  잡으면 두 마리가 함께 걷는다. (라운드 22초 기준 2.05라운드당 1회)
- **해금**: Lv N 처치 → Lv N+1 개방. 열린 레벨은 계속 재소환 가능 (낮은 레벨 = 안전하지만 보상 적음).
- **HP**: 1150 × 2.5^(Lv-1) · **장갑**: 3×Lv · 속도 26.
- **Lv3까지는 영웅·허수아비를 공격하지 않고 지나간다** — 저레벨 보스는 소득원이고,
  위협은 못 잡았을 때의 누출 라이프뿐. Lv4부터 잡몹 여러 기 몫으로 때린다.
- **보스는 영웅에게 저지되지 않는다** — 화력이 모자라면 일주를 마치고
  **라이프 2+Lv를 문다**. 도발 인형(허수아비 개조)만 보스를 붙잡을 수 있다.
- **레벨 사다리에 리스크가 있다** (항상 최고 소환 봇 측정): 처치율 Lv3 91% ·
  Lv4 73% · Lv5 54% · Lv6 75% — "항상 최고 소환 = 항상 승리"(구 96%)를 깼다.
- **Lv1 기준선** (테스트로 고정): 시작 미네랄 55(유닛 4기)로는 **2단계 조합이 떠야**
  잡는 판이고, 75(6기)면 거의 잡는다(≥85% — 웨이브 밀도 상향 후 완화된 문턱).
- 보스 처치는 영웅 XP의 페달이기도 하다 (xpPerBoss).

## 6. 영웅 — 골드→XP→자동 균등 성장 (3안, 2026-07-13)

```
영웅 파워 = 스탯(레벨업 자동 균등 성장) × 증강 배수(선택)
```

레벨 배수는 **폐지**됐다 — 골드 없이 XP만으로 오르던 ×2.4/레벨이 파워커브 과속
(Lv17 DPS 985 = Lv3 타워의 6배)의 원인이었다. 이제 영웅 파워는 전부 골드 비용을
가지므로 [income-curve.csv](../balance/income-curve.csv)에서 타워와 같은 저울에 오른다.

### 6.1 성장 한 축 — XP·레벨·스탯

- **만렙 Lv20 — 하드캡** (2026-07-21, 사용자 지시): 레벨업이 20에서 완전히 멈추고
  초과 경험치는 버려지며 XP 구매도 잠긴다(버튼·경험치 바가 만렙 상태를 표시).
  만렙 이후 영웅이 강해지는 길은 **증강 강화뿐**이다 — 레벨은 바닥을 깔고, 골드는
  등급(강화)으로 들어간다. 경험치 일시불(깨달음)은 만렙이면 드래프트에서 빠진다.
- **XP는 골드로 산다**: 3골드 = 1XP(버튼당 60골드=20XP, TFT식). 자연 성장(킬 XP 0.5/기,
  영웅 막타 ×2, 보스 8×Lv)만으로 R45 전후 만렙 — 구매는 **도달 시점을 앞당길 뿐**이다.
- **레벨업 = 세 스탯 자동 지수 성장** (2026-07-21, 사용자 지시: "스탯 지수그래프로"):
  `기본값 × 1.12^(레벨-1)` — 계단식 포인트 누적을 폐지했다. 하드캡이 있어 지수가
  안전하고, 만렙 직전 구간이 성장의 절정이 된다. 성장률 1.12는 힘 종점(2→17.2)이
  옛 곡선의 만렙 종점과 일치하는 값 — 공격력·체력의 만렙 앵커는 보존된다.
  민첩·지능은 종점이 커지는 만큼 공속 포화식과 SKILL_PER_INT(0.035→0.012)가 흡수한다.
- **만렙 공속 앵커**: 무증강 Lv20 = **1.96회/초** (사용자 지시 1.9~2.0,
  `LEVEL_ATTACK_SPEED_RATE` 0.005→0.045). Lv1은 0.73회/초로 거의 그대로다(보스 앵커
  보존). 무증강: Lv1 0.73 · Lv10 1.24 · Lv15 1.59 · Lv20 1.96. 공속은 마나 회전이기도
  해서 만렙 스킬 시전 빈도도 같은 비율로 빨라진다 — 만렙 영웅 DPS가 옛 곡선의 약 2배가
  되며, 공격 3증강 만렙 영웅이 GOD 한 기를 넘는다(옛 [미달 기록] 0.68배 → 1.9배 달성).
- 레벨업 스탯 선택과 선택 대기 UI는 없다. 빌드 방향 전환은 증강 선택으로 담당한다.
- 레벨업 비용: 14 × 1.08^L 지수 — 하드캡이 실제 상한이고, 지수는 캡을 푸는 실험을
  해도 경제가 고레벨을 못 사게 막는 안전망이다.
- 레벨업 시 완전 회복. 죽어 있으면 XP 구매 불가.

**기본 체력 260** (레벨 무관, 2026-07-14 신설). 체력이 `힘×70`뿐일 때 Lv1 영웅이 140이었고,
어그로 110에 몹이 10기씩 뭉쳐 붙어 **R1에 8.8초면 죽었다**(라운드는 22초). 측정: 시드 12판 중
9판에서 영웅 사망, 첫 사망 중앙값 R5. 기본값은 **초반만** 두껍게 한다 — 레벨이 오를수록
힘 몫이 커져 비중이 줄므로 후반 탱커 빌드의 존재 이유는 그대로다.

| 스탯 | 포인트당 효과 | 기본값 (Lv20 종점 = ×8.6) |
|---|---|---|
| 힘 | 공격력 +6 · 체력 +70 | 2 (→17.2) |

## 7. 증강 — 적응형 드래프트

영웅 타입/전직은 **없다**. 빌드 방향은 드래프트의 관성이 만든다.

> **임시 차단 (2026-07-19, 사용자 지시)**: 영웅 직접 캐리 축(공격 스탯·치명·즉사·평타
> 광역)은 드래프트에서 제외 중 — `data/hero.ts HERO_CARRY_BLOCKLIST`. 고점 예측이
> 어려워 밸런스를 못 잡는다는 판단. 영웅 역할은 **탱커·타워 보조**(어그로·방깎·감속·
> 화상 고정딜·경제·타워 강화)로 한정. 2026-07-21 정리로 스킬 강화(증폭·다중 투사)와
> 처치 스택 성장은 재설계 후 풀에 복귀했고, 회복·생존 5종(이지스·재기·최후의 저항·재생·
> 방벽)은 별도 비활성(`AUGMENT_DISABLED`)됐다. 데이터·효과 로직은 남아 있다 —
> 명단만 걷어내면 복구된다.

- **선택지**: 데이터 56종 · **실제 드래프트 33종** 풀에서 3장 (2026-07-21 대정리 —
  §11 결정 기록 참고). **가중치 = 1 + 0.9 × (그 계열 보유 수)** — 몰수록 그
  계열이 더 잘 뜬다. 강제가 아니라 관성: 첫 장부터 특화해도, 범용부터 집어도 된다.
- **한 축 한 장 원칙 (2026-07-21)**: 모든 카드는 실버로 뽑히고 **증강 강화**가 등급을
  올리므로, 같은 축의 변형 카드(A+B 조합·대가형·상위판)는 중복이다 — 축마다 한 장만 둔다.
- **계열 7종 — 기능축이다.** 계열 = 그 증강이 게임에 무엇을 더하는가.
  그래서 특화가 곧 플레이 스타일이 된다.

| 계열 | 하는 일 | 대표 (드래프트 가능 기준) |
|---|---|---|
| 강화 | 순수 수치 | (전부 차단 중 — 영웅 캐리 금지) |
| 방어 | 생존 — 피해 감소·흡혈·가시 | 중장갑, 흡혈, 가시 갑옷 |
| 특수 | 발동 효과 — 화상·감속·방깎·초신성 | 화염 부착, 서리 일격, 맹독 |
| 스킬 | **강화 5축** (피해·마나·투사체·범위·도트) + 스킬 디버프 | 증폭, 파장, 여파, 한파, 부식 |
| 성장 | 누적 — 처치·라운드마다 영구히 커진다 | 사냥 본능, 강철 혈통, 진화 |
| 경제 | 수입 — 지속(탐욕·정맥·학자) + **일시불**(골드·마정석·경험치) | 탐욕, 일확천금, 깨달음 |
| 유틸 | 기동·부활·타워 지휘·어그로·**타워 일시불** | 전쟁군주, 도발, 긴급 증원 |

**증강은 가산으로 쌓인다.** `공격력 +60%` 세 장은 +180%(×2.8)지 1.6³(×4.1)이 아니다.
시너지도 가산이다. 전부 곱이면 스택형 증강이 복리로 터진다 — 측정에서 성장 3장이
GOD 타워 **14.7기**(R44 보드 전체가 5.1기)까지 갔다.

**곱연산은 둘뿐이다** (`COMPOUNDING_IDS`): 광전사(최종 공격력 ×1.5) · 진화(성장 누적치 ×1.4).
곱이 흔해지면 다시 복리가 되므로, 소수만 두고 배율은 낮게 잡는다. 그래야 곱이 **사건**이 된다.

- **등급**: 실버 1.0× / 골드 2.0× / 플래티넘 3.5×. 드래프트는 **언제나 실버**(2026-07-20) —
  등급은 **증강 강화**(골드 소비)로만 오른다. 뽑기 운은 "무엇이 뜨는가"만 정한다.
- **강화 해금 = 만렙** (2026-07-21, 사용자 지시 — 당초 "모든 증강 획득 후"에서 변경):
  증강 강화는 **만렙(Lv20 하드캡) 도달 후**에 열린다. 만렙이 레벨 성장의 끝이자 강화의
  시작 — 레벨(바닥) → 증강(방향) → 강화(깊이)로 골드 싱크가 이어진다. 만렙 전에
  일시불 목돈으로 강화를 돌리는 연쇄도 같은 문으로 막힌다. 버튼은 잠금 사유(Lv n/20)를
  직접 보여준다.
- **일시불 증강** (2026-07-21): 일확천금(골드 **250**)·마정석 노다지(40)·깨달음(경험치
  **80**, 카드에 예상 레벨 표시)·긴급 증원(랜덤 타워)은 고르는 순간 지급된다. 일찍 뜰수록
  좋다 — **그것도 운이다** (사용자 결정). 값 앵커 [프로토]: 250 ≈ 중후반(R25·R35 픽)
  1.5라운드치 수입, 80XP ≈ 금화 환산 240.
  **강화도 일시금이되, 지급액이 강화 구매 비용에 묶인다** (`instantUpgradeGrant`):
  실버→골드 = 명목 비용×1.5, 골드→플래티넘 = ×1.0(본전). 무료 강화(R45)도 명목가 기준.
  노다지·깨달음은 같은 금액을 환율로 바꿔 받는다 (골드→XP는 XP 구매 환율 ÷3,
  골드→마정석은 [프로토] ×0.16). "일확천금 골드로 강화를 사서 또 일확천금이 강화되는"
  연쇄는 카드당 순이익 최대 +비용×0.5 한 번(그마저 강화 후보 2장 랜덤에 걸려야)으로
  끝나고, 전역 비용 카운터(×1.7)만 올려놓는다 — 루프가 성립하지 않는다.
  긴급 증원은 등급별 추가 타워를 즉시 받는다(실버 티어1×1 / 골드 +티어1×1 /
  플래티넘 +티어2×1, `TOWER_ROLL_BY_RARITY`) — 타워는 강화 비용보다 싸다.
- **흡혈은 등급 배수를 받지 않는다.** 가한 피해량 자체를 따라 성장하므로 다시 등급을 곱하면
  이중 성장한다. 카드별 기본 비율을 합산하되 최종 흡혈은 12%가 상한이다.
- **대가형(리스크) 증강**: 이득에 **대가**가 달린다 (예: 광전사 — 최종 공격력 ×1.5,
  최대 체력 -30%). **등급은 이득만 키우고 대가는 키우지 않는다** — 대가가 등급에 비례하면
  높은 등급이 뜬 게 손해인 경우가 생겨 "등급은 순수 운" 원칙이 깨진다.
- **리롤**: 카드마다 무료 1회 — 전체 3장이 아니라 각 카드만 바꾼다 (2026-07-20).
- **시너지**: 같은 계열 3개 = 특화, 5개 = 대특화. 후반 역전은 레벨이 아니라 여기서 나온다.
- **디버프 2×2** (2026-07-21): 평타 슬로우=서리 일격 / 평타 방깎=맹독 /
  스킬 슬로우=한파 / 스킬 방깎=부식. 축이 겹치지 않게 한 칸에 한 장.
- 제약: 스택 상한 / 대폭발은 충격파(광역) 필요 / 스킬은 하나만.
- 성장 스택은 **누가 잡았든** 모든 몹 처치를 센다 (2026-07-21, 막타 조건 폐지 — 값은 1/4로
  하향). '라운드마다 공격력' 성장은 폐지, 체력 성장(강철 혈통)만 남는다. 처형은 **보스에
  걸리지 않는다**. 폭사는 연쇄하지 않는다 — 폭발이 죽인 적은 다시 터지지 않는다.
- **화상은 평타로 안 붙는다 — 스킬과 도트(레이저 틱·불바다 장판)만 불을 붙인다.**
  중첩 상한이 없고, 겹당 피해는 **공격력 계수 없는 고정값**이며 **방어력을 무시한다(트루 피해)**.
  그래서 화염의 성장축은 "얼마나 세게"가 아니라 **"얼마나 여러 겹"**이다 —
  마나 회전이 빠를수록, 도트 틱이 잦을수록 두꺼워진다. 점화는 폭발이 아니라 **화염 피해 배수**다.
- 맹독은 딜이 아니라 **약화**다 — **방어력 감소**(타워 피해도 함께 이득).
  감속은 서리 일격의 축이다 (디버프 2×2, 2026-07-21).

## 8. 액티브 스킬 — 자동 시전

플레이어는 **무엇을 들고 어떻게 개조할지만** 고른다. 시전은 조건 충족 시 자동.

**보스 조준 규칙** (2026-07-21, 사용자 지시): 자동 시전·낙점의 밀집도 셈법에서

### 8.1 마나 — 쿨타임은 없다 (TFT식, 2026-07-14)

스킬은 **마나가 차면 자동으로 나간다.** 마나는 **평타를 칠 때(+10)와 맞을 때(+8)** 찬다.

- **공격 속도가 곧 스킬 회전이다.** 민첩에 투자하면 평타도 스킬도 빨라진다.
- **탱커도 스킬을 쓴다.** 맞으면 마나가 차므로 어그로를 끄는 것 자체가 자원이 된다.
  (`반격 집중` — 피격 시 마나 +12.)
- **채널링은 지속 중 마나가 차지 않는다.** 그래서 가동률에 천장이 생긴다 —
  레이저는 빔 3초 동안 마나가 멎고, 끝난 뒤부터 다시 찬다.
  단발은 마나가 차는 대로 계속 터진다. 그래서 "마나 증강을 몇 장 집을 것인가"의 답이
  스킬마다 다르다.
- 필요 마나 감소에는 바닥이 있다(원래 값의 40%). 없으면 '난사' 빌드가 같은 장수의
  강화 빌드를 10~40배로 압도했다 (측정 후 하향).

- **개조 2트랙**: 증강(질 — 폭발 화살, 연사, 도발 인형 등 질적 변화) +
  가스 구매(양 — 피해 +8%/회, 필요 마나 -6%/회, 비용 30×1.35ⁿ).
- **공용 개조**는 모든 스킬에 걸린다: 한파(스킬에 감속 추가) · 다중 투사(대상 +2, 튕김 +2) ·
  반격 집중(피격 시 마나 +12) · 마나 순환(획득 +18%) · 오버클럭(필요 마나 -18%, 피해 -12%) ·
  축전지(시전 후 마나 25 잔류).
- **레이저 개조**: 집속 렌즈(두께) · 증폭 코일(길이·지속) · 고속 조사(도트 간격 0.5→0.25초).
  **장판 개조**: 넓은 화선(반경·지속) · 맹렬한 불길(초당 피해).

   [설계 문서](../exec-plans/finite-ending-design.md). 웨이브 재설계(§2.1)로 무적 중반은
   해소됐고 봇 사망이 R55~66에 온다. 남은 것: **R60 최종 보스·종결 연출**(지금은 그냥
   계속 어려워질 뿐 "끝"이 없다), R51~60 증강 차단, **증강 레벨 재배치**(5·6번째

   목표 생존곡선(R45 30%·R60 클리어 1%)은 사람 플레이테스트로 확정.

## 14. 밸런스 시트 관리 방침

- **원본은 코드다** (`web/src/data/*.ts`). 사람이 시트를 고치지 않는다.
- `npm run gen:balance` → `docs/balance/*.csv` 7종 자동 생성.
  `tests/balance-csv.test.ts`가 코드-시트 일치를 강제한다 (어긋나면 CI 실패).
- 외부 공유(구글 시트 등)는 **읽기 전용 사본**으로 올린다. 시트에서 수치를 바꾸고
  싶으면 코드를 바꾸고 재생성한다 — 역방향 동기화는 하지 않는다.
