# 세계관 — 중세 판타지

> 상태: **현행** · 최종 갱신: 2026-07-15
> 이 문서는 배경·용어·표현 방향의 기준이다. 전투 규칙과 수치는
> [게임 시스템 명세](game-system-spec.md), 유닛별 수치는 코드를 따른다.
> 2026-07-13의 초한지 대체역사 설정은 2D·3D 영웅 레퍼런스 탐색 결과 제작성이 낮아
> 이 방향으로 **대체됐다**.

## 1. 한 줄 설정

왕국의 변경에 있는 마력 요새에서, 플레이어는 정체를 알 수 없는 군세의 반복되는
공세를 막으며 여러 종족과 길드의 전력을 임시로 규합한다.

특정 역사나 유럽 국가를 재현하지 않는 **창작 중세 판타지**다. 익숙한 기사·마법·괴물의
문법을 사용하되, 세부 역사와 국가 설정은 실제 게임 콘텐츠가 필요할 때만 확정한다.

## 2. 플레이어와 영웅

플레이어는 요새의 지휘관이다. 난수로 도착하는 지원 전력을 합성하고, 직접 전선을 오가는
영웅의 빌드를 선택해 공세를 버틴다.

- 타워는 병사 한 명이 아니라 **초소·병기·마법 구조물·소환 진지**다.
- 합성은 같은 전력 두 개를 더 강한 임의 전력으로 재구성하는 게임적 표현이다.
- 영웅은 타워 사이를 오가는 유일한 전투 인물이며, 첫 영웅은 **방랑기사**다.
- 방랑기사는 어느 왕국에도 속하지 않은 용병 기사다. 요새에 고용되어 전선을 붙들지만,
  출신·본명·성별은 콘셉트 아트와 플레이테스트 전까지 확정하지 않는다.

## 3. 표현 범위

- 갑옷, 성벽, 길드, 룬, 연금술, 마법 병기와 다양한 판타지 종족을 허용한다.
- 화풍은 사실적 중세가 아니라 **스타일라이즈드 로우폴리 판타지**를 기준으로 한다.
- 화면을 덮는 장식보다 55° 부감에서 읽히는 큰 형태와 역할 구분을 우선한다.
- 잔혹하고 암울한 다크 판타지보다 모험적이고 명료한 톤을 쓴다.
- 실제 게임에 등장하지 않는 왕국사·신화·종족사는 미리 만들지 않는다.

## 4. 네 병과 (왕국군)

플레이어의 모든 타워는 하나의 **왕국군**에 속하고, 코드의 `race` 4종은 세력이 아니라
왕국군의 **네 병과**로 표현한다. 정통 인간 전력(정규군·마법대)에 더해 바깥의 힘을
불러 싸운다 — 정령·골렘을 **소환**하고, 공성 병기를 **제작**한다. 전투 태그
(파워·스플래시·스피드)는 병과와 직교하는 별도 축이다.

| 병과 | 코드 `race` | 성격 | 태그 무게중심 |
|---|---:|---|---|
| 정규군 | 0 | 인간 병사·궁수·기병 | 스피드 |
| 포병 | 1 | 기계식 공성 병기 (화약 없음 — 노포·투석·충차) | 파워 |
| 마법대 | 2 | 마탑 원소·룬·마검 | 스플래시 |
| 소환대 | 3 | 소환한 정령·골렘·구조물 | 혼합 |

병과별로 강화한다(파일런 업그레이드). 티어당 유닛 수가 곧 계급 라인 수다 — 정규군·포병·
마법대는 2 라인(예: 활/기사), 소환대는 1 라인. 유닛 이름은 티어가 오를수록 초라함→
아이코닉으로 오른다(견습 궁병→신궁→발키리, 미니 골렘→콜로서스). 개별 유닛명은 순수
플레이버라 밸런스와 분리되며 코드의 `engine/src/data/units.ts`가 원본이다([백로그](backlog.md)에 리네이밍 기록).

## 5. 등급과 최상위 전력

Lv1~Lv4는 구조물의 크기, 무기 수, 마력 효과와 실루엣 밀도로 상승을 표현한다.
`GOD`의 최종 표시 명칭은 미확정이다. 초한지 설정의 실재 명장 규칙은 폐기하고,
각 병과를 대표하는 고유 전력으로 만든다. 프로토 명칭은 `units.ts`에 있고 최종 표시
명칭은 플레이테스트 후 확정한다.

- 정규군: 팔라딘·발키리 등 전설의 전사
- 포병: 왕립 대노포·공성 거탑 등 전설의 공성 병기
- 마법대: 리치·원소 군주 등 대마도사
- 소환대: 아이언 콜로서스 등 명장 골렘

고유 인물을 타워에 사용할 경우 영웅과 구분되도록 단 위에 고정하고, 큰 깃발이나 구조물을
붙인다. 이 원칙은 기존 부감 가독성 판단을 유지한 것이다.

## 6. 적 — 야생군

적은 **야생군** — 길들여지지 않은 야수·괴물이다(플레이어의 소환대·조련 전력과 대비).
보병형 괴물, 돌진형 야수, 거대 보스를 기본 문법으로 삼아 코드의 세 유형과 연결한다.

| 코드 | 표현 | 시각적 역할 |
|---|---|---|
| `normal` | 고블린·언데드 등 보병형 괴물 | 작은 실루엣의 무리 |
| `hunter` | 늑대·마수·암흑 기사 등 돌진형 적 | 빠르고 날카로운 실루엣, 영웅 위협 |
| `boss` | 거대 괴수·적 지휘관 | 큰 질량과 고유 실루엣 |

구체적인 적 종족과 라운드별 배치는 모델 프로토타입 이후 결정한다.

## 7. 아트 디렉션

### 기준 이미지와 사용 범위

| 이미지 | 역할 | 사용 범위 |
|---|---|---|
| [외부 화풍 레퍼런스](../reference/kris-cooper-bran-the-breaker-kris-cooper.jpg) | 북유럽 장식화의 선·면·색감 참고 | **화풍 참고 전용.** 인물·문장·늑대·구도·테두리는 복제하지 않는다 |
| [방랑기사 v1](../reference/wandering-knight-concept-v1.png) | 영웅의 비율·재질·팔레트 기준 | 이후 인간형 영웅의 기준 앵커 |
| [방랑궁수 v1](../reference/wandering-archer-concept-v1.png) | 같은 세계의 경갑·활·가죽 표현 기준 | 원거리 영웅과 목재 무기의 기준 앵커 |

외부 레퍼런스는 출하 리소스가 아니다. 생성물은 외부 이미지의 특정 인물이나 문장을
재현하지 않고 아래의 추상적인 조형 규칙만 공유한다. 새 콘셉트를 만들 때는 외부 이미지
단독보다 **방랑기사·방랑궁수 v1을 함께 참조**해 프로젝트 내부 일관성을 우선한다.

### 공통 화풍 DNA

- **매체:** 중세 필사본 장식화의 평면성과 스타일라이즈드 로우폴리 게임 조형의 결합
- **선:** 짙고 선명한 외곽선, 안쪽 경계는 한 단계 가늘게. 사진 같은 부드러운 경계는 피한다
- **면:** 복잡한 광택보다 넓고 평평한 색면 3~5개로 대상을 먼저 읽히게 한다
- **질감:** 종이·과슈처럼 약한 입자, 닳은 금속·가죽·목재. 표면 노이즈가 실루엣을 이기지 않는다
- **팔레트:** 차콜, 슬레이트 블루, 낡은 은색, 짙은 갈색, 이끼 회색을 기본으로 한다
- **강조색:** 한 리소스당 하나. 천·깃발·룬·무기 효과 중 가장 넓게 보이는 한 부위에만 둔다
- **분위기:** 무겁고 오래된 장비지만 공포·고어보다 영웅적이고 명료한 모험 판타지
- **문양:** 단순한 매듭·기하 문양은 허용. 실제 가문 문장이나 외부 레퍼런스의 상징은 쓰지 않는다

### 형태

- 캐릭터와 전력은 머리·어깨·무기 비율을 과장한다.
- 얇은 장식, 긴 끈, 복잡한 갑옷 겹침보다 큰 덩어리 3~5개로 실루엣을 만든다.
- 계열과 등급은 색만이 아니라 무기와 구조물 형태로 구분한다.
- 영웅은 이동하는 인간형 실루엣, 타워는 바닥에 고정된 구조물로 즉시 구분되어야 한다.
- 최종 판정은 확대 일러스트가 아니라 **Unity 55° 부감의 실제 표시 크기**에서 한다.

### 색과 재질

- 철, 가죽, 목재, 석재의 물성색 위에 역할별 강조색을 사용한다.
- 마법 발광을 허용하되 화면 전체가 네온처럼 보이지 않도록 핵심 부위에 제한한다.
- 유닛은 현행 가독성 측정대로 보드보다 밝게 유지한다.
- 병과 식별색은 무기 효과, 깃발, 천, 룬처럼 부감에서 보이는 넓은 면에 싣는다.

### 리소스별 생성 규격

| 종류 | 반드시 보이는 것 | 피할 것 |
|---|---|---|
| 영웅 | 머리·어깨·주무기의 고유 실루엣, 큰 재질 덩어리 | 몸을 가리는 망토, 여러 무기, 얇은 장식 다발 |
| 타워 | 바닥 받침, 공격 방향, 계열별 핵심 무기 | 사람만 덩그러니 세우기, 직육면체에 색만 바꾸기 |
| 일반 적 | 이동 방향과 머리 형태, 무리에서 읽히는 단순 외곽 | 갑옷 세부 묘사, 가는 팔다리, 타워와 같은 받침 |
| 사냥꾼 | 앞으로 쏠린 자세, 날카로운 전방 실루엣 | 일반 적과 같은 직립 자세, 둔중한 정사각형 체형 |
| 보스 | 일반 적보다 넓은 발자국, 한 가지 거대 특징 | 작은 장식만 늘려 등급 표현하기 |
| 소품 | 재질과 용도를 나타내는 한 가지 큰 특징 | 캐릭터보다 높은 대비·발광, 플레이 상태색과 충돌 |

타워는 `받침 + 몸체 + 핵심 무기 + 계열 표식`의 네 덩어리로 설계한다. 상위 티어는
장식의 개수가 아니라 높이·무기 크기·상부 외곽의 변화로 구분한다. 생성형 2D 시안에서도
이 네 덩어리를 분리해 보여야 Meshy와 수작업 모델링 양쪽에서 재구성이 쉽다.

### 생성 실행은 resources에서 관리한다

공통 프롬프트 골격, 종류별 추가 문구, Meshy 입력용 콘셉트 규격, 리깅(Mixamo) 전략,
생성 후 판정 체크리스트는 **아트 방향이 아니라 실행·산출물**이므로 이 문서에 두지 않는다.
생성 프롬프트와 원본 이미지는 [`resources/`](../../resources/README.md)에서 관리한다 —
이 문서는 위의 화풍 원칙만 기준으로 유지하고, resources는 그 원칙을 실제 프롬프트로 옮긴다.

### 첫 영웅 콘셉트

- 낡은 판금과 가죽 장비를 섞은 무소속 용병 기사
- 큰 양손검 또는 한손검과 방패 중 하나만 사용
- 비대칭 견갑 하나와 짧고 두꺼운 망토로 실루엣을 만든다
- 왕실 문장과 화려한 금장 대신 닳은 금속, 수선 자국, 작은 개인 표식을 사용한다
- 얼굴 묘사보다 투구·어깨·무기가 55° 부감에서 읽히는지를 우선한다
- Meshy 실험을 위해 A-pose, 분리된 팔, 단색 배경의 정면 이미지를 먼저 만든다

방랑궁수는 같은 인물의 무기 교체가 아니라 같은 세계에 속한 별도 원거리 영웅의 시각 기준이다.
판금 비중을 줄이고 가죽·천·목재를 늘리되, 얼굴·체형을 반드시 방랑기사와 같게 만들 필요는 없다.

무기 구성과 얼굴 노출 여부는 콘셉트 시안 비교 후 확정한다.

## 8. 외부 애셋 정책

이 저장소는 공개이므로 커밋하는 외부 애셋은 **CC0만** 사용한다. 중세 판타지 전환으로
Kenney·Quaternius의 유럽풍 저폴리 소품도 문화 필터 없이 검토할 수 있지만, 실제 라이선스와
원본 재배포 조건은 애셋마다 확인한다. Unity Asset Store 에셋은 무료여도 원본 파일을 공개
저장소에 커밋하지 않는다.

맵 타일은 상태를 색으로 표시하므로 절차적 생성을 유지한다. 외부 애셋은 판 바깥 소품과
검증용 임시 전력에 우선 사용한다.

## 9. 용어 기준

세계관 전환으로 초한지 전용 용어인 `성주`, `지원병 모집`, `재편`, `병과 훈련`,
`명장`, `적장`, `군자금`, `군량`, `보급대`는 현행 기준에서 제거한다.
`종족`은 **병과**(정규군/포병/마법대/소환대)로 코드·UI에 반영한다(§4). 나머지 기존
용어는 UI 전면 개편 전까지 유지한다.

| 시스템 용어 | 현재 표시 기준 | 향후 후보 |
|---|---|---|
| 타워 | 타워 | 전력 / 진지 |
| 생성 | 소환 | 모집 / 건설 |
| 합성 | 합성 | 강화 / 재구성 |
| 종족 | **병과** (정규군·포병·마법대·소환대) | 진영 / 길드 |
| GOD | GOD | 전설 / 신화 |
| 보스 | 보스 | 우두머리 |

병과명 외 나머지 후보는 구현·인지 테스트 전까지 UI 명세가 아니다.

## 10. 현재 확정된 것과 남은 것

### 확정

- 초한지 대체역사에서 창작 중세 판타지로 전환한다.
- 스타일라이즈드 로우폴리와 부감 실루엣 가독성을 기준으로 삼는다.
- 첫 영웅의 원형은 무소속 용병인 방랑기사다.
- 방랑기사·방랑궁수 v1을 이후 리소스 생성의 내부 아트 기준으로 사용한다.
- 인간 군세로 제한했던 적을 판타지 괴물과 적 지휘관으로 확장한다.
- 외부 애셋의 CC0 정책과 절차적 맵 타일은 유지한다.
- `race` 4종을 왕국군 단일 팩션의 **4 병과**(정규군/포병/마법대/소환대)로 재편하고,
  유닛을 티어=계급 사다리로 리네이밍한다([프로토], 원본은 `units.ts`).

### 미확정

- 세계와 왕국의 이름, 역사, 종족 구성
- 개별 유닛의 **최종 표시 명칭** (프로토는 `units.ts`에 반영, 플레이테스트 후 확정)
- 최상위 등급의 표시 명칭과 고유 전력
- 방랑기사의 본명·성별·얼굴·무기 구성
- 적 세력과 라운드별 세부 로스터
