## 1. 목적과 범위

지금 게임의 타워는 **단일 텍스처 2종**(`tower` 28×28 / `towerGod` 32×32)을 race 색으로
`setTint`만 바꿔 전부 같은 벽돌 포탑으로 그린다([`sprites.ts:230`](../../phaser/src/sprites.ts)).
Lv/GOD 라벨 숫자와 색·스케일만 다르다.

이 문서는 그것을 **계열별 고유 도트**로 바꾸기 위한 컨셉을 정리한다. 핵심 원칙:

1. **계열 통일성** — 한 계열(Lv1→Lv4)은 "같은 병사가 장비를 업그레이드하는" 것처럼 보인다.
   재질·실루엣의 뼈대(spine)를 공유하고, 티어마다 무기·갑옷·구조물이 좋아진다.
2. **초라함 → 아이코닉** — 세계관 §4대로 Lv1은 성글고 초라하게, Lv4는 촘촘하고 위풍당당하게.
   상승은 **실루엣 밀도·무기 수·규모·마광**으로 읽힌다(세계관 §5).
3. **병과 즉시 구분** — 55°가 아닌 측면 도트라도 실루엣 아키타입만으로 병과가 읽혀야 한다
   (궁수 / 기사 / 세로 병기 / 가로 병기 / 지팡이 로브 / 궤도 오브 로브 / 석골렘).
4. **GOD은 독립 영웅** — 계열의 연장이 아니라 각 병과를 대표하는 일점 전력. 왕관·후광·큰 캔버스로
   "타워가 아니라 강림한 존재"로 보이게 한다(세계관 §5, [사용자 지시: GOD은 독립적으로]).

**범위 밖:** 태그(파워/스플래시/스피드) 표식은 도트에 넣지 않는다 — 이미 투사체 VFX가
그 채널을 말한다([`projectile-fx.ts`], 메모리: 이펙트 절제·이미 말하는 채널엔 추가 금지).
수치·밸런스도 이 문서 밖이다.

## 2. 공통 도트 문법

### 2.1 병사 + 진지 이중 구조

세계관 §4에서 타워는 "병사 한 명이 아니라 초소·병기·마법 구조물·소환 진지"다. 하지만 사용자가
원하는 "장비가 업그레이드되는 병사"를 함께 살리려면 각 타워를 **[전투 인물] + [발판/병기/진지]**의
한 덩어리로 그린다:

| 병과 | 인물 | 진지/병기 |
|---|---|---|
| 정규군 | 궁수·기사 병사 (인물이 주역) | 목책 사대 / 방벽 초소 (보조) |
| 포병 | 조작수 (보조) | 투석기·노포 (병기가 주역) |
| 마법대 | 로브 마법사 (인물이 주역) | 룬진·부양 결정 (마광 보조) |
| 소환대 | — | 소환된 크리쳐 자체가 주역 + 발밑 소환진 |

계열 통일성은 **주역 요소의 재질 spine**으로, 상승은 인물 장비 + 진지 규모 **양쪽**으로 표현한다.

### 2.2 상승을 읽히는 4개 축 (세계관 §5)

- **실루엣 밀도**: Lv1 성글다(뚫린 곳 많음) → Lv4 촘촘(장식·부속 가득)
- **무기 수·크기**: 활 1 → 리컬브, 볼트 1 → 회전 탄창, 부양 룬 0 → 3+
- **규모**: 받침·프레임·소환진이 커진다
- **마광**: 마법대·룬 계열은 티어마다 발광 픽셀이 늘고 밝아진다

### 2.3 색 파이프라인 (§7의 결정 대상)

현행은 **그레이 5단계 + 런타임 단일 tint**다. 이 방식으론 "나무활 + 강철갑옷 + 천옷"의
재질 대비를 한 화면에서 못 낸다(전부 한 색으로 물듦). 권고안은 §7에 정리한다.

## 3. 계열 컨셉 시트 — 정규군 (race 0 · 강철슬레이트 `#6d86ab`)

### 3.1 궁병 계열 (speed)

`army-apprentice-archer → army-archer → army-longbowman → army-master-archer`

**Spine:** 활을 든 궁수 실루엣 + 등 화살통 + 목책 사대. 재질 기조 = **천·가죽 + 나무활**.

| 티어 | 이름 | 인물·장비 | 진지 |
|---|---|---|---|
| Lv1 | 견습 궁병 | 천 두건, 짧은 나무 반곡궁, 화살통 없음, 가죽신 — 초라 | 낮은 목책 한 겹 |
| Lv2 | 궁병 | 가죽 흉갑 + 한쪽 어깨 견갑, 표준 활, 등 화살통, 손목 브레이서 | 목책 사대 |
| Lv3 | 장궁병 | 사슬셔츠, 키만 한 장궁, 깃 화살 다발, 삼각 어깨망토 | 석축 발판 |
| Lv4 | 신궁 | 부조 갑주 + 망토·깃, 은장식 리컬브 컴포짓 보우, 조준 시 룬빛 화살 | 깃발 달린 망루 |

계열 표식: **손에 든 활 + 등 화살통**은 Lv1부터 유지. 활이 짧은 나무 → 긴 장궁 → 리컬브로 진화.

### 3.2 기사 계열 (speed)

`army-squire → army-knight → army-sword-expert → army-sword-master`

**Spine:** 검 + 방패 근접 병사 + 방벽 초소. 재질 기조 = **판금 + 천 서코트**.

| 티어 | 이름 | 인물·장비 | 진지 |
|---|---|---|---|
| Lv1 | 종자 | 누비옷 + 철모(kettle), 짧은 검, 둥근 목방패 — 갑옷 최소 | 목책 초소 |
| Lv2 | 기사 | 판금 흉갑 + 바이저 투구, 롱소드, 카이트 실드, 서코트 | 총안 있는 석벽 |
| Lv3 | 소드 엑스퍼트 | 풀 플레이트, 양손검, 문장 방패, 어깨 망토 | 깃발 성벽 |
| Lv4 | 소드마스터 | 부조 흑철 갑주, 룬 각인 대검, 나부끼는 망토, 투구 깃 — 아이코닉 | 첨탑 초소 |

궁병과의 구분: **정면 방패 + 세운 검**(근접) vs **당긴 활**(원거리) 실루엣.
화풍 기준은 [방랑기사 컨셉아트](../reference/wandering-knight-concept-v1.png)의 풍화 강철 + 갈색 가죽 톤.

## 4. 계열 컨셉 시트 — 포병 (race 1 · 청동 `#bf7a3a`)

병사보다 **병기가 주역**. 조작수는 크기 대비로 규모를 읽히는 보조. 재질 기조 = **목재 + 청동 + 로프**.
두 계열은 **실루엣 방향으로 즉시 구분**한다 — 투석기 = **세로로 던지는 팔**(곡사), 노포 = **가로 볼트 축**(직사).

### 4.1 투석기 계열 (power)

`artillery-small-catapult → artillery-mangonel → artillery-trebuchet → artillery-megalith-catapult`

| 티어 | 이름 | 구조 |
|---|---|---|
| Lv1 | 소형 투석기 | 작은 나무 팔 + 비틀림 로프 + 돌멩이, 바퀴 없음, 조작수 1 |
| Lv2 | 망고넬 | 큰 팔 + 청동 보강 + 슬링 주머니, 나무 바퀴 |
| Lv3 | 트레뷰셋 | 높은 A자 프레임 + 긴 팔 + 평형추 상자 + 청동 힌지 |
| Lv4 | 거석 투석기 | 거대 이중 프레임 + 쇠사슬 평형추 + 거석 탄 + 청동 윈치, 조작수 2, 깃발 |

### 4.2 노포 계열 (power → Lv4 폭풍)

`artillery-small-ballista → artillery-ballista → artillery-heavy-ballista → artillery-repeating-ballista`

| 티어 | 이름 | 구조 |
|---|---|---|
| Lv1 | 소형 노포 | 작은 나무 활대 + 짧은 볼트 + 삼각대 |
| Lv2 | 발리스타 | 청동 보강 활대 + 홈통 + 크랭크 윈치 + 방패막 |
| Lv3 | 중발리스타 | 이중 두꺼운 활대 + 큰 볼트 거치 + 철 프레임 |
| Lv4 | 연발 발리스타 | 회전 볼트 탄창(리볼버식) + 청동 톱니 + 조작수 — 다연장 |

계열 표식: 수평 활대 + 볼트가 Lv1부터 유지, 활대가 두꺼워지고 탄창화한다.

## 5. 계열 컨셉 시트 — 마법대 (race 2 · 자수정 `#9a6ea6`)

로브 마법사 + 마력 구조물. 재질 기조 = **로브 + 룬 + 부양 결정 + 자수정 마광**.
두 계열 구분 — 마법사 = **지팡이 정면 캐스팅**, 정령술사 = **몸 주위를 도는 원소 궤도 오브**.

### 5.1 마법사 계열 (splash)

`magic-apprentice-mage → magic-mage → magic-sorcerer → magic-archmage`

| 티어 | 이름 | 인물·마구 |
|---|---|---|
| Lv1 | 견습 마법사 | 짧은 천 로브 + 두건, 나무 지팡이 + 작은 수정 끝 — 초라 |
| Lv2 | 마법사 | 정식 로브 + 어깨술, 룬 각인 지팡이 + 부양 오브 1 |
| Lv3 | 마도사 | 장식 로브 + 깃, 큰 결정 지팡이 + 부양 룬 3, 눈빛 마광 |
| Lv4 | 아크메이지 | 별 장식 로브 + 뾰족 후드, 뜬 결정 클러스터 + 손끝 마법 + 룬 후광 |

### 5.2 정령술사 계열 (splash)

`magic-apprentice-elementalist → magic-elementalist → magic-elemental-sorcerer → magic-grand-elementalist`

| 티어 | 이름 | 인물·원소 |
|---|---|---|
| Lv1 | 견습 정령술사 | 소박 로브, 손 위 작은 원소 불씨(오브 1) |
| Lv2 | 원소술사 | 로브 + 토템 지팡이, 몸 주위 회전 원소 오브 2 |
| Lv3 | 원소 마도사 | 원소 룬 갑주 + 오브 3 + 발밑 마법진 |
| Lv4 | 대원소술사 | 부양 원소 코어 + 정령 날개형 마력 + 큰 마법진 + 뿔 왕관 |

계열 표식: 마법사는 **지팡이 축**, 정령술사는 **궤도 오브 개수**가 티어를 읽힌다.
(냉기 계열 이펙트가 붙는다면 눈꽃 고정 의미만 채택 — 메모리: 에셋 선별.)

## 6. 계열 컨셉 시트 — 소환대 (race 3 · 이끼 `#8a9a5b`)

1 라인. **소환된 크리쳐 자체가 타워** — "장비 업그레이드"가 아니라 "소환체 격 상승"으로,
통일성은 **돌·룬 재질 + 이끼/룬 발광**으로 잡는다. 발밑 **소환진**이 진지성을 준다(티어마다 커짐).

### 6.1 골렘 계열

`summon-mini-golem → summon-gargoyle → summon-magma-golem → summon-rune-golem`

| 티어 | 이름 | 소환체 |
|---|---|---|
| Lv1 | 미니 골렘 | 작은 바위 골렘, 이끼 낀 팔다리, 외눈 |
| Lv2 | 가고일 | 날개 달린 석상 — 라인 내 변주(비행 실루엣), 돌 날개 + 뿔 |
| Lv3 | 매그마 골렘 | 균열 사이 용암 빛 큰 골렘, 갈라진 바위 피부 |
| Lv4 | 룬 골렘 | 룬 각인 정제 석조 거상, 부양 룬석 관절, 이끼 → 룬빛 |

> **주의(원작 근거):** 가고일은 다른 3티어와 실루엣이 튄다(날개·비행). 이는 원작 풀 인덱스 구조가
> 그리 묶은 것이라 의도로 수용하되, "돌 + 발광" 재질과 소환진으로 계열 소속을 읽힌다.

## 7. GOD 티어 — 독립 영웅 (11종 · 32×32)

계열의 연장이 아니다. 각 병과를 대표하는 **일점 전력**이라 독립 실루엣 + 받침 강림 후광 +
왕관/오라로 "타워가 아니라 강림한 존재"로 그린다. 병과색은 **강조 트림**으로만 쓴다(전신 tint 금지).
초기 풀(보스 5처치 이하 · early)과 확장 풀(6처치 이상 · late)을 구분한다([`units.ts`](../../engine/src/data/units.ts) GOD_POOL).

| 병과 | GOD | 해금 | 컨셉 |
|---|---|---|---|
| 정규군 | 발키리 | early | 날개 달린 여전사, 창·방패, 빛나는 갑주 |
| 정규군 | 팔라딘 | late | 성기사 대검 + 빛 오라 + 망토 + 후광 |
| 포병 | 공성 거탑 | early | 바퀴 달린 이동 공성탑, 다수 병사, 사다리 |
| 포병 | 왕립 대노포 | late | 황금 장식 거대 발리스타 + 왕실 문장 |
| 포병 | 파멸의 투석기 | late | 흑철 프레임 + 화염 거석 탄 |
| 포병 | 폭풍 노포 | late | 뇌전 감긴 볼트 노포 |
| 마법대 | 리치 | early | 언데드 대마법사, 해골 + 부양 로브 + 영혼불 + 왕관 |
| 마법대 | 대현자 | late | 노현자, 부양 서적·룬 성좌 + 긴 수염 + 지팡이 |
| 마법대 | 원소 군주 | late | 불·얼음·뇌 3원소 코어 융합 + 날개 |
| 소환대 | 아이언 콜로서스 | early | 리벳·증기 철제 거상, 압도적 규모 |
| 소환대 | 미아즈마 로드 | late | 독무 정령 군주, 부유 촉수 + 독안개 |

공통 GOD 문법: (a) 발판에 강림 후광(부드러운 글로우, 이미 있는 `glow` 텍스처 재사용),
(b) 개별 실루엣, (c) Lv 타워보다 큰 캔버스(32×32↑), (d) 라벨은 'G'.

## 8. 컨셉아트 생산 플랜 (1단계)

도트를 먼저 굽지 않는다. **먼저 컨셉아트로 계열의 룩을 확정**하고, 승인된 아트를 기준 삼아
도트로 변환한다. 아래는 아트를 발주·생성할 때 쓰는 스펙이다(이 문서는 이미지를 만들지 않는다).

> **바로 붙여넣는 생성 프롬프트:** 아래 §8.1~8.2 스펙을 외부 이미지 AI용 완결형 프롬프트로 옮겨
> [`docs/reference/towers/`](../reference/towers/README.md)에 정리했다 — 계열 진화 시트 7종 +
> GOD 11종. 계열별: [정규군](../reference/towers/prompts-lines-army.md) ·
> [포병](../reference/towers/prompts-lines-artillery.md) ·
> [마법대](../reference/towers/prompts-lines-magic.md) ·
> [소환대](../reference/towers/prompts-lines-summon.md) ·
> [GOD](../reference/towers/prompts-god.md).

### 8.1 화풍 앵커 (고정)

기존 레퍼런스가 이미 정한 룩을 계열 전체로 확장한다 — 새로 발명하지 않는다:

- **스타일**: 치비 로우폴리 판타지 (2~3등신), 각진 로우폴리 페이스팅 + 소프트 셀 셰이딩.
  근거: [방랑궁수 치비](../reference/wandering-archer-chibi-tpose-concept-v1.png),
  [방랑기사](../reference/wandering-knight-concept-v1.png).
- **재질 톤(D1 하이브리드의 기준색)**: 풍화 강철 회청 · 브라운 가죽 · 미색 천 · 흑철 다크.
  켈틱 트라이케트라 각인 모티프는 정규군 한정 소품으로만.
- **프레이밍**: 정면 T포즈 턴어라운드 1장 + 3/4 액션 포즈 1장. 무배경(회색), 그림자 없음 —
  도트 추출이 쉽게.
- **금지**: 다크판타지 과잉·화면 덮는 장식(세계관 §3), 태그 표식(D3).

### 8.2 시트 구성 (발주 단위)

| 시트 | 내용 | 목적 |
|---|---|---|
| 계열 진화 시트 ×7 | 한 계열의 Lv1→Lv4를 좌→우 4단 나열 | "장비가 좋아진다"는 통일성 검수 |
| GOD 시트 ×11 | 개별 영웅 1장씩 (턴어라운드 + 액션) | 독립성·강림감 검수 |
| 병과 라인업 ×4 | 병과 내 전 계열을 한 판에 | 실루엣 아키타입 구분 검수 |

계열 진화 시트가 **최우선** — 통일성과 초라함→아이코닉 곡선을 여기서 판정한다.

### 8.3 도트 변환 사양 (2단계, 아트 승인 후)

컨셉아트가 승인된 뒤에만 픽셀 캐릭터로 변환한다. 픽셀 캐릭터는 게임에서 기본 방향을 읽히게 하는
**앞면 1종 + 뒷면 1종**을 우선 생성한다. 두 방향은 같은 캐릭터의 장비·색·SD 비율을 공유해야 하며,
방향별 크기와 발 기준점도 맞춘다.

애니메이션 제작은 이 문서와 저장소의 기본 생성 범위에서 제외한다. 공격 모션·이동 모션·방향별
이미지 시퀀스·GIF/스프라이트 시트는 외부 애니메이션 툴 또는 사이트에서 제작하고, 게임에 넣을 때
필요한 파일 형식과 프레임 규격만 별도 통합 단계에서 검토한다.

현행 [`sprites.ts`](../../phaser/src/sprites.ts) 규약을 계승해 확장한다:

- **텍스처 키**: `tower`/`towerGod` 단일 → **유닛 id별 키**(예: `t:army-apprentice-archer`).
  `BattleScene.syncTowers`와 [`TowerPreviewScene`](../../phaser/src/scenes/TowerPreviewScene.ts)가
  id로 텍스처를 고르게 바꾼다. 도트 정의는 데이터 테이블(id → draw 함수)로 관리.
- **캔버스 크기**: Lv 타워 28×28 유지 검토, 부족하면 32×32(D2). GOD은 36×36까지(보스 균형).
- **방향**: 기본 캐릭터는 앞면·뒷면 2종을 우선 제작한다. 애니메이션 제작 도구에서 추가 방향이
  필요할 때 이 두 이미지를 기준 레퍼런스로 삼는다.
- **격리 프리뷰 우선**: 본 게임에 바로 넣지 않는다(메모리: 격리 프리뷰 후 통합).
  `TowerPreviewScene`/별도 랩에서 기본 캐릭터의 실루엣·비율·투명 배경을 확인한 후 통합한다.

### 8.4 결정 현황

| # | 결정 | 상태 |
|---|---|---|
| D1 | **색 파이프라인** — 단일 tint vs 고정 팔레트 | **확정(2026-07-23): 하이브리드.** 받침·진지·계열 배너는 병과색 tint 유지, 인물·무기·재질(나무 갈색·강철 회청·천 미색)은 고정 소팔레트로 그려 재질 대비를 살린다. 병과색은 배너·트림·발광 액센트로 병과를 읽힌다 |
| D2 | 캔버스 크기 상향(28→32) 여부 | 미확정 — 도트화 단계에서 28로 먼저 시도, 디테일 부족 시 32 |
