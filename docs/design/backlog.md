# 백로그 — 미확정 아이디어

> 상태: **현행** (수시 갱신) · 최종 갱신: 2026-07-13
> 명세에는 구현·결정된 것만 들어간다. 여기는 아이디어의 대기실이다 —
> 채택되면 명세로, 기각되면 사유와 함께 아래 "기각" 절로 옮긴다.

## 시스템 후보

- **GOD 타워 타입 변경 (미네랄 리롤)** — GOD 타워를 클릭해 미네랄로 종류를 다시 뽑는다.
  사용자 제안(2026-07-10). **새 미네랄 싱크**라 후반 미네랄 잉여 구조와 영웅 XP 구매 곡선에
  영향을 준다. 채택하려면 `web/src`에 먼저 구현 → `sim:ga`로 미네랄 소비 곡선·XP 구매 횟수
  변화를 측정 → 근거가 서면 명세에 기록 후 Unity 미러링.
  UI 자리는 [HUD 재설계 계획](../exec-plans/unity-hud-redesign.md) §4에서 비워 둔다.
- **웨이브 타입 시스템** (2026-07-12 제안, 아이디어 단계). 5R 사이클에 질적 정체성을 —
  기본 / 돌격(빠름·HP↓) / 중장갑(느림·HP↑) / 비행(어그로 면역 — 보스 통과 메커니즘 일반화) /
  사냥꾼(공격력 높음·영웅 위협 전담, R10+ 5의 배수 혼입). 기본 몹 공격력은 대폭 하향해
  위협을 타입에 위임한다. 코어 루프의 "웨이브 예측" 결정에 내용이 생기고 태그·크리쳐
  카운터플레이가 산다.
  **제약**: ①총체력 예산 보존 배수로 설계(속도는 노출시간을 줄여 실질 난이도↑ — 시뮬 필수)
  ②타입 시각 구분 필수(지금 몹은 전부 같은 구슬) ③HUD에 다음 웨이브 예고.
  **착수안**: 골격 + 타입 2개(기본·사냥꾼)로 출발, 나머지는 데이터 행 추가.
- **빙고/미션 시스템 — 수입의 두 번째 결정 축** (2026-07-11 승격). 원본의 보유 미션류
  (예: "티어1 유닛 전종 보유 시 골드"). [income-curve.csv](../balance/income-curve.csv)가
  드러낸 근거: 웨이브 보상·킬 마일스톤은 결정론이라 **수입 격차를 만드는 선택이 보스 레벨
  하나뿐이다** (R17 기준 저점 739 vs 고점 1,174 — 전부 보스 차이). 미션은 "빌드를 특정
  방향으로 틀면 골드"라는 두 번째 축이 된다 — 조합 운을 소득 기회로 바꿔주는 부수 효과도.
  채택 시 원본 미션 목록을 맵 분석에서 재확인 후 설계.
- **초월 티어 / 점수 연장전** — GOD 위 단계, 또는 R60 이후 BTD6 프리플레이식 점수 모드.
  원본 §11.4의 `God LvNN` 49종이 힌트. **유한 종결([설계](../exec-plans/finite-ending-design.md))을
  택했으므로 당장은 불필요** — R60이 끝이다. 무한 리플레이를 원하면 그때 검토.
- **유닛 생성 비용 곡선** — 12미네랄 고정이 타일 포화(R61)와 GOD 양산(R51에 20기)을
  동시에 만든다. 누적 증가로 바꾸는 안. **제약: 초반에 미네랄이 없어 할 게 없어지면 안 된다.**
  근거: [생존곡선 진단](../reports/survival-curve-diagnosis-v0.1.md) §2·§3.
- **몹 총 체력 곡선을 플레이어 DPS 곡선에 정합** — 지금은 플레이어가 S자, 몹이 지수라
  중반 위험률 0%·후반 80%의 계단이 생긴다. 사이클 HP 점프 구조를 연속 곡선으로 바꾸는 안.
- **원거리 레인 보강** — 관통, 일제 사격 계열 증강 확률 상향.
  근거: 원거리 레인 최약 (R80, 영웅몫 19%).
- **중반 영웅 공백 처방** — R30~45에 영웅몫 0~10%. 후보 3안:
  중반 몹 HP↑·수↓ / 영웅 선공 어그로 / 보스 어그로 영웅 우선.
- **보스 리스크 확장** — 저지 불가로 리스크가 생겼지만 GA에서 보스 유전자
  신호가 여전히 약함. 보상·페널티 확대 검토.
- **후반 영웅 특훈(직접 스탯 구매)** — 지금은 도입하지 않는다. 영웅 스탯은 레벨에 따라
  자동 균등 성장하고 미네랄 영웅 싱크는 XP 구매가 담당한다. R45 이후 미네랄 잉여와
  XP의 한계효용 저하가 시드 시뮬·플레이테스트에서 함께 확인될 때만 후반 전용 싱크로 검토한다.
